Conditional Statements / Control Flow

In the last section, you learned about expressions and operators. Now, we’ll see how to combine these with conditional statements / Control Flow to make your applications more interactive. Conditional statements allow your program to make decisions based on certain conditions. In JavaScript, there are two main types: if...else and switch...case.

Example Scenario

Imagine you’re building an app where you need to greet users based on the time of day. Here’s how it will work:

  • If the hour is between 6 am and 12 pm, we’ll say, “Good Morning.”
  • If it’s between 12 pm and 6 pm, we’ll say, “Good Afternoon.”
  • For any other time, we’ll say, “Good Evening.”

To create this type of logic in your application, we use if...else statements.

Structure of if...else

The structure of an if...else statement is simple. Here’s how it works:

  • if (condition) { ... }: If the condition is true, the code inside { ... } will run.
  • else if (another condition) { ... }: If the first condition is false but this condition is true, this code will run. You can have as many else if statements as needed.
  • else { ... }: If none of the previous conditions are true, the code here will run. This is optional.

Note: When you have only one line of code, the { ... } brackets can be left out.

Writing the Code

Here’s how we would write this greeting example in code: 

let hour = 10; // Example hour value

if (hour >= 6 && hour < 12) {
 console.log("Good Morning");
} else if (hour >= 12 && hour < 18) {
 console.log("Good Afternoon");
} else {
 console.log("Good Evening");
}
In this example:
  • hour >= 6 && hour < 12 checks if the hour is between 6 am and 12 pm.
  • hour >= 12 && hour < 18 checks if the hour is between 12 pm and 6 pm.
  • The else statement handles all other cases.

This code will print "Good Morning" if hour is 10, "Good Afternoon" if hour is 14, and "Good Evening" for any other time outside those ranges.

Overview
  • if checks the first condition.
  • else if adds more conditions if the first one is false.
  • else is the default action if none of the conditions are met.

This is how we use if...else to add logic to our applications, making them smarter and more interactive!

switch…case

Imagine you want to check the role of a current user, which could be a guest, moderator, or admin. While you could handle this using multiple if...else statements, there's another way to manage it more neatly: by using switch...case.

How switch...case Works

1. Starting the Switch Statement:

  • Begin with the switch keyword, then add parentheses () and put the variable you’re checking (like role) inside them. This variable will be checked against different values in the switch statement.
switch (role) {}
2. Adding Cases:
  • Inside the switch block (within {}), you can define different cases to handle each possible value of role.
  • Each case is like a test: it compares the variable’s value to a specific option.
case 'guest':
 console.log("Guest User");
 break;
  • This checks if role is 'guest'. If it is, it will print "Guest User". Then, the break keyword stops the program from continuing to the next case. Without break, the code will keep running through all cases, even if they don’t match, which we usually don’t want.
3. Using the Default Case:
  • If none of the case values match the role, you can use default to run code for any unknown role:
default:
 console.log("Unknown User");

The default case is optional but helpful for catching unexpected values

Here’s a complete example: 
let role = 'guest';
switch (role) {
 case 'guest':
 console.log("Guest User");
 break;
 case 'moderator':
 console.log("Moderator User");
 break;
 default:
 console.log("Unknown User");
}
In this example:
  • If role is 'guest', it will print "Guest User".
  • If role is 'moderator', it will print "Moderator User".
  • If role has any other value, it will print "Unknown User".

Comparison with if...else

You can achieve the same logic using if...else:

if (role === 'guest') {
 console.log("Guest User");
} else if (role === 'moderator') {
 console.log("Moderator User");
} else {
 console.log("Unknown User");
}

Both switch...case and if...else do the same thing. switch...case can be a bit cleaner for multiple specific values, while if...else may feel simpler.

Loops / Iterations

Sometimes, we need to repeat the same task multiple times. For example, if we want to display "Hello World" five times in the console, we could write this: 

console.log("Hello Naseeb");
console.log("Hello Naseeb");
console.log("Hello Naseeb");
console.log("Hello Naseeb");
console.log("Hello Naseeb");

This works, but it’s not efficient. There’s a better way to do it that’s where loops come in. Loops let us repeat an action without having to write the same code over and over.

In JavaScript, we have different types of loops, like:
  • for loop
  • while loop
  • do...while loop
  • for...in loop
  • for...of loop

All these loops essentially do the same thing they help us repeat an action a specific number of times. The difference between them is mainly in how they start and end. Here, we’ll focus on the for loop.

For Loop

To print "Hello World" five times using a for loop, we can write:

for (let i = 0; i < 5; i++) {
 console.log("Hello World");
}
This code will display "Hello World" five times. Let's break down how it works:
  1. Start with for: A for loop starts with the keyword for.
  2. Parentheses (): Inside the parentheses, we set up three parts:
    • Initialization: We declare a variable, let i = 0;, where i is a counter that starts at 0.
    • Condition: We set a condition, i < 5;, which keeps the loop going as long as this condition is true.
    • Increment: We increase i by 1 each time the loop runs with i++.
  3. Curly Braces {}: Inside the curly braces, we put the code we want to repeat. In this case, it’s console.log("Hello World");.

Step-by-Step

Here's how this loop runs:

  1. Initialization: i is set to 0.
  2. Condition Check: It checks if i < 5. If true, the code inside the braces runs.
  3. Execute and Increment: After printing, i increases by 1 (increment). The loop checks the condition again.
  4. Loop End: Once i reaches 5, the condition becomes false, and the loop stops.

While Loop

In the original for loop:

for (let i = 0; i < 5; i++) {
 console.log("Hello World");
}
  • We start with i set to 1.
  • The loop will run as long as i is less than or equal to 10.
  • Each time through the loop, it checks if i is odd (i.e., i % 2 !== 0). If i is odd, it logs the value of i.
  • After each loop cycle, i is increased by 1 (i++).

Translating to a while Loop

To write this in a while loop, we need to declare i separately since, in a for loop, the variable i is defined within the loop itself. This difference is a key feature of how while loops and for loops work.

Here’s the while loop version: 
let i = 0;
while (i < 5) {
 console.log(“Hello World”);
 i++;
}
Key Points to Understand
  1. Variable Declaration: In a while loop, the loop variable i is defined outside the loop (let i = 0;). It’s not a part of the loop structure itself, so we manage it manually.
  2. Loop Condition: The while loop runs as long as i < 5 is true. The condition is checked at the start of each loop cycle.
  3. Loop Increment: Inside the while loop, we increment i manually with i++ at the end of each loop. If we forget this, the loop could run indefinitely.
  4. Scope: In the for loop, i is only accessible within the loop. In a while loop, i was declared outside the loop, so it can still be used outside the loop if needed.

do…while loop

The do...while loop is the third type of loop in JavaScript. It’s very similar to the while loop, but there’s one key difference. In a do...while loop, the code inside the loop will run at least once, even if the condition is false.

Here’s how the do...while loop works:
  • You start by declaring any variables outside the loop.
  • The do keyword begins the loop and is followed by a block of code inside {} braces
  • After the code block, the while keyword checks if the condition is true.
  • If the condition is true, the loop runs again. If it’s false, the loop stops.
Let’s rewrite a while loop as a do...while loop.

With the while loop:

let i = 0;
while (i < 5) {
 console.log("NaseebCodeStudio");
 i++;
}

With the do...while loop:

let i = 0;
do {
 console.log(“Hello World”);
 i++;
 }
while (i < 5);

Key Difference

In the do...while loop, the code inside {} will run at least once, no matter what. This happens because the condition is checked after the code block runs. 

let i = 9;
do {
 console.log(i);
 i++;
} while (i <= 5);

Here, even though i starts at 9 (which is greater than 5), the code will run once and print 9 before the condition is checked. If this were a while loop, it wouldn’t run at all because the condition isn’t true from the start.

When to Use do...while

In real projects, we don’t use do...while loops very often. Most of the time, for or while loops are more practical. But it’s important to know the difference and understand that do...while guarantee at least one run of the code inside, which can be useful in some specific situations.

This explanation simplifies the process and makes it easy to see how do...while differs from other loops!

Infinite loops

An infinite loop is a loop in programming that keeps running forever because it never meets the condition to stop. Accidentally creating an infinite loop can make your program freeze or even crash your computer. Here’s a simple breakdown of how it happens and examples to help you avoid it.

What is an Infinite Loop?

An infinite loop is a loop that never ends because the exit condition is never met. For example, if you’re using a loop to count numbers from 1 to 5 but forget to tell it when to stop, it will keep counting forever. This can cause problems like making your browser spin endlessly, slowing down your system, or even freezing your computer.

Example of an Infinite Loop in JavaScript

Here’s a simple example of how an infinite loop can happen:

let i = 0;
while (i < 5) {
 console.log(i);
}

In this example, i is set to 0, and the condition says to keep running while i is less than 5. But if we forget to increase i (like i++), it stays at 0 forever, causing the loop to keep running without end.

To fix it, add i++ inside the loop: 
let i = 0;
while (i < 5) {
 console.log(i);
 i++;
}

Other Common Infinite Loop Examples

1. Using true in the Condition: This type of loop will never stop since true is always true

while (true) {
 console.log("This will run forever");
}

This loop will run endlessly. To stop it, add a condition or break statement within the loop.

2. Infinite Loop in do...while:

let i = 0;
do {
 console.log(i);while (true);
}

Similar to while (true), this loop will keep running since the condition true never changes.

3. Infinite for Loop:

for (let i = 0; i < 10; ) {
 console.log(i);
}

Here, we forgot to increment i, so it will stay the same, causing the loop to run forever.

Why Avoid Infinite Loops?

Infinite loops take up a lot of system resources, which can slow down or crash your browser or computer. If you find your program or browser is frozen due to an infinite loop, you might need to force close it using the Task Manager (Ctrl + Shift + Esc on Windows) or by closing the tab or browser window.

for…in Loop

For...in loop: We have learned about three types of loops in JavaScript: for, while, and do...while. These loops are used to repeat an action as many times as needed. However, there are two additional types of loops specifically for iterating over the properties of objects or elements in an array: the for...in loop and the for...of loop.

Let's print all the properties of an object using the for...in loop. Here's an example: 

const person = { 
  name: "Naseeb", 
  age: 24 
};

Using for...in loop:

With the for...in loop, we still use the for keyword and parentheses, but we don’t have the typical initialization, condition, and increment parts as we do with other loops. Instead, it looks a little different: 

for (let key in person) {
 console.log(key);
}

In each iteration, the key will hold one of the properties of the person object. For example, with the person object above, the output will be: name and age.

What if we want to print the value of each property next to its key? We already know that there are two ways to access an object's property: using dot notation (e.g., person.name) or bracket notation (e.g., person['name']).

Why Use Bracket Notation in for...in Loop?

When we don't know ahead of time which property we’ll access, we use bracket notation because the property name is determined at runtime. Therefore, in the for...in loop, we can't use dot notation, and we must use bracket notation to display each property’s value. Here’s the corrected code:

for (let key in person) {
 console.log(key, person[key]);
}

This is the correct way to use the for...in loop.

Using for...in with Arrays

Although we can use for...in with arrays, it is not recommended. Here’s an example with an array: 

const colors = ['red', 'green', 'blue'];
for (let index in colors) {
 console.log(index);
}

In this loop, index represents each index number of the colors array. The output will be: 0, 1, and 2. If we want to get the element at each index, we can use bracket notation as follows: 

for (let index in colors) {
 console.log(index, colors[index]);
}

As we learned in the last lecture, starting from ECMAScript 6 (or ES6), we have a new way to iterate over an array, which is by using the for...of loop. It is very similar to the for...in loop, but instead of the “in” keyword, we use the “of” keyword.

In each iteration of the for...of loop, the color variable represents each element of the array, so we get 'red', 'green', and 'blue' directly. We use the for...in loop to iterate over the properties of an object, while for...of loop is used to iterate through the elements of an array."

Nested Loop

In JavaScript, we can use loops inside other loops (nested loops) to make our code more efficient and concise, depending on our program's needs. For example, if we want to print the multiplication table of 2, we could write the following code with complete syntax: 

for (let n = 1; n <= 10; n++) {
 console.log("2 x " + n + " = " + n * 2);
}

If we want to print tables 3, 4, 5, 6... up to 10, we would have to write this code multi times. However, by using a nested loop, we can make our code shorter and more efficient. The outer loop can iterate over each number for which we want the table, and the inner loop can handle the multiplication for each table.

Here’s how we could print the tables from 1 to 10:

for (let n = 1; n <= 10; n++) {
 console.log("Table of " + n);
 for (let nn = 1; nn <= 10; nn++) {
 console.log(n + " x " + nn + " = " + n * nn);
 }
 console.log(""); // Adding a blank line for better readability between tables
}

OR
for (let n = 1; n<= 10; n++){
for (let nn = 1; nn<=10; nn++){
console.log(n+ ' x '+ nn + ' = ' + n * nn);
}
console.log("");
}

Explanation of How the Nested Loop Works

1. Outer Loop (for (let n = 1; n <= 10; n++)):

  • This loop iterates from 1 to 10, representing each number for which we want to print a multiplication table.
  • For each iteration of n, the inner loop will execute 10 times, printing the full table for the current value of n.
2. Inner Loop (for (let nn = 1; nn <= 10; nn++)):
  • The inner loop handles the multiplication for each table. For each value of “n” (from the outer loop), “nn” will iterate from 1 to 10.
  • During each inner loop iteration, console.log(n + " x " + nn + " = " + n * nn); prints a line of the table, showing n multiplied by “nn” with the resulting product.

For example, when n is 2:

  • The inner loop runs with “nn” from 1 to 10, producing the multiplication table for 2 (2 x 1 = 2, 2 x 2 = 4, ..., 2 x 10 = 20).
  • After completing 10 iterations of the inner loop, the outer loop moves to the next value, n = 3, and the inner loop runs again to print the table for 3, and so on.

By using nested loops, we efficiently print each table from 1 to 10 without having to repeat our code for each number

Break and Continue

You have learned about loops. Now, there are two keywords used with loops to change their behavior: break and continue. These keywords can be used with any loop, and each serves a specific purpose in controlling loop execution.

Break Keyword

Sometimes, we need to jump out of a loop for specific reasons or conditions that occur during runtime. In such cases, we use the break keyword to exit the loop immediately.

Example: Suppose we have a loop that increments a variable, “I”, and we want to stop the loop when “I” reaches 5.

let i = 0;
while (i < 9) {
 if (i === 5) break;
 console.log(i);
 i++;
}

In this example, when “I” becomes equal to 5, the break keyword will terminate the loop. Therefore, the output will include values from 0 to 4, and the loop stops when it reaches 5.

Continue Keyword

The “continue” keyword is used when, at a certain point in a loop, you want to skip the current iteration and move directly to the next one without completing the remaining code in that iteration. This is useful when you want to "jump over" a certain condition.

Example: Suppose we want to print only the odd numbers from 1 to 10. We can use the “continue” keyword to skip even numbers. 

let i = 0;
while (i <= 10) {
 if (i % 2 === 0) {
 i++;
 continue;
 }
 console.log(i);
 i++;
}

In this example, when “I” is an even number, it increments and moves to the next iteration, effectively skipping the even numbers in the output. Therefore, the console will display 1, 3, 5, 7, and 9.

When the JavaScript engine encounters the “continue” keyword, it jumps to the beginning of the loop, bypassing any remaining statements in that iteration. For example, when “I” is 2, it's even, so i++ makes “I” 3. The loop then skips directly to the next iteration without executing console.log(i), resulting in the display of only odd numbers.

Write a program that takes two numbers and returns the maximum of the two.

Solution: 
let number = max(1, 2);
console.log(number);
function max(a, b) {
 if (a > b) return a;
 return b;
}

In this example, if any argument is greater, our function will work as expected. To test this function with different arguments, we can call it with various test cases. First, let's assume the second argument is greater. Then, we assume the first argument is greater. Finally, we test if both arguments are equal.

When writing code or functions, it's important to test different values to ensure the function behaves as expected. However, this code can be improved for better readability and good coding practice.

Improving the Code

  1. Removing the else Keyword: Since the return statement exits the function immediately, there is no need to use the else keyword. We can simply return “a” if a > b; otherwise, return b.
  2. Using a Ternary Operator: We can make this code even simpler by using a ternary operator. This way, the function returns the maximum value in a single line.
Here's the improved code: 
function max(a, b) {
 return (a > b) ? a : b;
}
Or more improved for true and false: 
function max(a, b) {
 return (a > b);
}

Now, our function is more concise and still works correctly. This version has only the return statement, making it a cleaner and more efficient solution. In this case the return keyword always return the value of condition which is going to be Boolean like true or false so no need to use “a : b” from ternary operator.

Exercise 2

Write a program to check if an image is in landscape orientation or not. This is similar to the previous exercise you just learned. 

function isLandscape(width, height) {
 if (width > height) return true;
 return false;
}

In this example, we check if the width is greater than height. If it is, the function returns true, indicating a landscape image. Otherwise, it returns false.

However, we just learned that we could use a ternary operator for this: 
(width > height) ? true : false;

Both approaches are functionally the same. However, writing code like this is considered very basic and amateurish. We don’t want to explicitly return true or false in this way, as it makes the code less readable. Instead, we can simplify the program by removing the explicit true and false return values.

Improved Version

In JavaScript, a conditional expression like (width > height) evaluates directly to true or false. So, we don’t need to explicitly write return true or return false. The function can be written simply as: 

function isLandscape(width, height) {
 return (width > height);
}

This version will automatically return true if width is greater than height (indicating a landscape orientation) or false otherwise. There’s no need to manually specify the return values of true and false, making the code more concise and professional.

Testing the Function

Let's test this function with a console log to confirm it works as expected: 

console.log(isLandscape(800, 600)); // Expected output: true

If we change 800 to 300, making the image a vertical orientation, we should get false: 

console.log(isLandscape(300, 600)); // Expected output: false

This approach makes the code cleaner and more efficient, without any unnecessary parts.

Exercise 4

This exercise is a very popular interview question called “FizzBuzz Algorithm." The goal is to create a function that takes an input and returns a specific output as a string based on certain conditions:

  1. If the number passed as input is divisible by 3, we get "Fizz."
  2. If the number is divisible by 5, we get "Buzz."
  3. If the number is divisible by both 3 and 5 (like 15), we get "FizzBuzz."
  4. If the number is not divisible by either 3 or 5, we get the same number as output.
  5. If we don't pass a number (e.g., we pass a string or other non-numeric data), we should get the message: "Not a Number."

These are the five types of output generated by this program. Now, let’s see the solution. Change the value according to your choice in the fizzBuzz parentheses (this will act as an argument to store your input). 

const output = fizzBuzz(5);
console.log(output);
function fizzBuzz(input) {
 if (typeof input !== 'number') return 'Not a Number';
 if (input % 3 === 0) return 'Fizz';
 if (input % 5 === 0) return 'Buzz';
 if (input % 3 === 0 && input % 5 === 0) return 'FizzBuzz';
 return input;
}

In this program, initially, you may encounter an issue were passing 15 only returns "Fizz" instead of "FizzBuzz." This happens because the if statement for input divisible by 3 is placed before the one checking divisibility by both 3 and 5. To fix this, we moved the condition (input % 3 === 0 && input % 5 === 0) to the top of the function

You can also replace the string "Not a Number" with the special JavaScript value NaN to represent non-numeric input, like so: 
function fizzBuzz(input) {
 if (typeof input !== 'number') return NaN;
 if (input % 3 === 0 && input % 5 === 0) return 'FizzBuzz'; // corrected
 if (input % 3 === 0) return 'Fizz';
 if (input % 5 === 0) return 'Buzz';
 return input;
}

In JavaScript, NaN stands for "Not a Number" and can be used instead of the string "Not a Number" in your program.

In JavaScript, certain values are considered falsy:

  • null
  • " " (Empty String)
  • False
  • 0
  • NaN

All other values are considered truthy.

When we call this function, it should return the number of truthy values in the array passed to it. 

const array = [0, null, undefined, ‘’, 1, 2, 3];

function countTruthy(array) {
 let count = 0;
 for (let value of array) {

 }
 return count;
}

  4. Return Result: return count; Returns the total count of truthy values.

This function will now correctly count and return the number of truthy values in any array passed to it.
