Operators

In JavaScript, operators are symbols or keywords that perform actions on values (like numbers or variables) to create expressions. These expressions are used to implement logic or algorithms. Here’s a simple breakdown of the different types of operators in JavaScript:

Arithmetic Operators:

Used to perform basic math operations like addition (+), subtraction (-), multiplication (*), and division (/).

Assignment Operators

These operators are used to assign values to variables. For example, = is used to assign a value, while += adds a value to an existing one.

Comparison Operators

Used to compare two values, such as checking if one is greater than the other (>), equal (==), or not equal (!=).

Logical Operators

These operators help in combining multiple conditions, such as && (and), || (or), and ! (not).

Bitwise Operators

These work with numbers at the binary level (0s and 1s), performing operations like AND (&), OR (|), and XOR (^).

Using these operators, you can write code that performs calculations, makes decisions, and solves complex problems in JavaScript.

Arithmetic Operators

Arithmetic operators are symbols used in programming to perform basic math calculations, just like we do in real life. Let’s go through them step by step:

Addition Operator (+)

This operator adds two numbers together

let a = 20;
let b = 30;
console.log(a + b); // Output: 50

Here, a + b is an expression that adds 20 and 30, producing the result 50

Subtraction Operator (-)

This operator subtracts one number from another.

let a = 20;
let b = 10;
console.log(a - b); // Output: 10

Multiplication Operator (*)

This operator multiplies two numbers.

let a = 10;
let b = 20;
console.log(a * b); // Output: 200

Division Operator (/)

This operator divides one number by another. 

let a = 100;
let b = 10;
console.log(a / b); // Output: 10

Remainder (Modulus) Operator (%)

This operator divides two numbers and returns the remainder (what’s left over).

let a = 10;
let b = 3;
console.log(a % b); // Output: 1

Here, 10 divided by 3 leaves a remainder of 1.

Exponentiation Operator (**)

This operator raises one number (the base) to the power of another number.

let a = 2;
let b = 4;
console.log(a ** b); // Output: 16

This means 2 raised to the power of 4, which equals 16.

Unary Operators

Increment Operator (++)

This operator adds 1 to a number.

let a = 10;
a++;
console.log(a); // Output: 11

+"5" // converted to number

After a++, the value of a becomes 11

Decrement Operator (--)

This operator subtracts 1 from a number.

let a = 10;
a--;
console.log(a); // Output: 9

After a--, the value of “a” becomes 9.

Prefix and Postfix

For increment (++) and decrement (--) operators, you can use them before or after the variable

  • Prefix (++a or --a): The operator changes the value before it’s used in the expression.
  • Postfix (a++ or a--): The operator changes the value after it’s used in the expression.
  • Increment and decrement are only applied on a variable not a number
let a = 10;
console.log(++a); // Output: 11 (prefix, a is increased first)
console.log(a++); // Output: 11 (postfix, a is increased after this line the previous addition is
included)
console.log(a); // Output: 12

Assignment Operators

Assignment operators are symbols used to assign values to variables. We have already seen a common assignment operator:

let a = 10;

In this example, the = (equal sign) is an assignment operator. It assigns the value 10 to the variable a

Other Assignment Operators

There are many other assignment operators that make it easier to update the value of a variable. Let’s look at some of them:

Increment Operator (++):

This adds 1 to a variable. For example:

n++; // This is the same as writing: n = n + 1;

The value of n is read, increased by 1, and then stored back in n.

Addition Assignment Operator (+=):

If you want to add a number other than 1, for example 5, you could write:

n = n + 5;

But there’s a shorter way using the += operator:

n += 5;

This adds 5 to the current value of n

Subtraction Assignment Operator (-=):

To subtract a number from n, you can use:

n -= 5;

This subtracts 5 from n.

Multiplication Assignment Operator (*=):

To multiply n by a number, use: 

n *= 5;

This multiplies n by 5.

Division Assignment Operator (/=):

To divide n by a number, use:

n /= 5;

This divides n by 5.

Exponentiation Assignment Operator (=)**:

If you want to raise n to the power of a number, use

n **= 5;

This raises n to the power of 5 (n = n^5).

Remainder Assignment Operator (%=):

To get the remainder of n divided by a number, use:

n %= 5;

This calculates the remainder when n is divided by 5.

Comparison Operators

Comparison operators are used to compare two values. As the name suggests, these operators check if one value is larger, smaller, equal, or not equal to another value. The result of using comparison operators is always a Boolean value, meaning the outcome is either true or false.

Types of Comparison Operators

There are two main types of comparison operators:

Relational Comparison Operators

These compare the size or order of two values.

Less than (<): Checks if the value on the left is smaller than the value on the right

let a = 10;
let b = 20;
console.log(a < b); // true

Greater than (>): Checks if the value on the left is larger than the value on the right.

console.log(a > b); // false

Less than or equal to (<=): Checks if the value on the left is smaller than or equal to the value on the right.

console.log(a <= b); // true

Greater than or equal to (>=): Checks if the value on the left is larger than or equal to the value on the right

console.log(a >= b); // false

Equality Comparison Operators

These operators check if two values are equal or not, with a focus on both value and data type.

Lose Equality or Equal (==): Checks if the values on both sides are the same, regardless of data type.

console.log(a == b); // false, it checks only value not type
console.log(1 == 1); // true
console.log(‘1’ == 1); // true
console.log(true == 1); // true, it automatically change the right side to true

Strict equal (===): Checks if both the value and the data type are the same.

console.log(a === b); // false, it checks the value + type

Not equal (!=): Checks if the values are different.

console.log(a != b); // true

Strict not equal (!==): Checks if the value or data type are different. 

console.log(a !== b); // true

These operators help in decision-making in programming by comparing values and giving you a true or false outcome based on the comparison.

Ternary or Conditional Operators

A ternary operator, also known as a conditional operator, allows you to simplify if-else statements in your code. It is a quick way to check a condition and return one value if the condition is true and another value if it's false, all in one line

Example of Ternary Operator

Let’s say you want to mark customers as gold or silver based on their points. If a customer has more than 100 points, they should be marked as a gold customer. Otherwise, they are marked as silver.

Here’s how you can do that with a ternary operator:

let points = 110;
let type = points > 100 ? 'gold' : 'silver';

Another Example:
let marks = 90;
let result = marks >= 90 ? "A Grad" : marks >= 80 ? "B" : marks >= 70 ? "C" : "Pass";
  • points > 100 creates a boolean value (true or false).
  • The question mark (?) checks if the condition is true.
    • If true, it assigns 'gold' to the type variable.
    • If false, it assigns 'silver'.

In simple terms:

  • The code starts with a condition (points > 100)
  • The question mark (?) checks if that condition is true.
  • If true, the first value ('gold') is used.
  • If false, the value after the colon ('silver') is used.

This is a ternary operator because it involves three parts: condition, true value, and false value. It’s a compact way of writing an if-else statement!

Logical Operators

Logical operators in JavaScript help us make decisions based on multiple conditions. There are three main logical operators:

  1. Logical AND (&&)
  2. Logical OR (||)
  3. Logical NOT (!)

Let's break down how each of these works with easy-to-understand examples.

Logical AND (&&)

The && (AND) operator returns true only when both conditions on the left and right sides are true. If either or both are false, it returns false.

console.log(true && true); // Output: true
console.log(true && false); // Output: false
console.log(false && true); // Output: false
console.log(false && false); // Output: false

In the second example, since one condition is false, the result is false. The && operator looks for at least one false to return false.

Where to use it:

Imagine we are building a loan approval app. To approve a loan, the applicant must have both a high income and a good credit score. If both are true, the loan will be approved.

let highIncome = true;
let goodCreditScore = true;
let eligibleForLoan = highIncome && goodCreditScore;
console.log(eligibleForLoan); // Output: true

Logical OR (||)

The || (OR) operator returns true if either condition on the left or right side is true. Even if only one condition is true, it will return true. If both conditions are false, only then will it return false.

console.log(true || false); // Output: true
console.log(false || false); // Output: false
console.log(false || true); // Output: true
console.log(true || true); // Output: true

In the first example, one condition is true, so the result is true. The || operator looks for at least one true to return true

Where to use it:

Using the same loan approval example, if either the applicant has a high income or a good credit score, they can still be approved for a loan.

let highIncome = false;
let goodCreditScore = true;
let eligibleForLoan = highIncome || goodCreditScore;
console.log(eligibleForLoan); // Output: true

Even though the income is low, the applicant has a good credit score, so they are still eligible.

Logical NOT (!)

The ! (NOT) operator simply reverses the value of a condition. If the condition is true, it makes it false, and if it's false, it makes it true.

console.log(!true); // Output: false
console.log(!false); // Output: true

Where to use it:

Let’s say we want to reject the loan application if the applicant is not eligible. We can use the ! operator to reverse the result of eligibleForLoan.

let highIncome = false;
let goodCreditScore = false;
let eligibleForLoan = highIncome || goodCreditScore;
let applicationRefused = !eligibleForLoan;
console.log(applicationRefused); // Output: true

In this case, since the applicant doesn't qualify for the loan (eligibleForLoan is false), we use ! to reverse it, and the loan application is refused (true).

Overview:

  • && (AND): All conditions must be true for the result to be true.
  • || (OR): At least one condition must be true for the result to be true.
  • ! (NOT): Reverses the condition true becomes false, and false becomes true.

These operators are essential for making decisions in your code based on multiple conditions!

Logical Operators with non-Boolean values

In JavaScript, logical operators like (|| and &&) don’t always return just true or false like in many other languages. Instead, they return the actual value of the operands, which makes them very powerful.

Example 1: 
console.log(false || true); // true

Here, false || true returns true because one of the values is true.

Example 2: 
console.log(false || "Naseeb"); // "Naseeb"
Example 3: 
console.log(false || 1); // 1

In this case, the result is 1 because 1 is a truthy value (it’s not zero or false).

Key Concept: Truthy and Falsy

  • Falsy values are undefined, null, 0, false, "" (empty string), NaN (not a number), -0, 0n, and document.all.
  • Truthy values are anything that is not falsy.

console trick to find truthy and falsy values by using !!. 

In Console:
!!"Naseeb Khan" // True
!"Naseeb Khan" // False

So, when using logical operators, JavaScript evaluates each value. If it finds a truthy value, it returns that value, not just true or false.

Short-circuiting

If you have multiple values in an expression like:

console.log(false || 1 || 2); // 1

Here, JavaScript starts with false, then moves to 1 (which is truthy), and stops there, returning 1. This process is called short-circuiting because as soon as a truthy value is found, JavaScript stops checking the rest.

Real-life example:

Imagine an app where users can pick a color for a t-shirt. If the user doesn’t select a color, you want to use a default color:

let userColor = 'red';
let defaultColor = 'blue';
let currentColor = userColor || defaultColor;
console.log(currentColor); // red

Here, the user selected "red", so the result is "red".

If the user didn’t choose a color:

let userColor = undefined;
let defaultColor = 'blue';
let currentColor = userColor || defaultColor;
console.log(currentColor); // blue

Now, since userColor is undefined (a falsy value), it returns the default color "blue".

In short, logical operators in JavaScript can handle more than just true or false. They work with truthy and falsy values and stop evaluating as soon as they find a truthy value. This feature allows for flexible code, like setting default values when no user input is provided.

Operator Precedence

When we work with complex expressions, we have to know the "precedence" or "priority" of different operators. Operator precedence tells us the order in which operations are performed in a mathematical expression. If we don’t understand this, we might get unexpected results.

Here’s an example to illustrate: 
let a = 2 + 2 / 2;

What do you think the result of this expression is? If you guessed 3, you’re correct! Here’s why:

In JavaScript (and many other programming languages), operators have different levels of precedence. The division (/) has a higher precedence than addition (+). This means the division part will be calculated first, not the addition.

To understand the order, we can use a rule called BODMAS:

  1. B: Brackets – solve anything in brackets first.
  2. O: Orders (exponents, powers).
  3. D: Division.
  4. M: Multiplication.
  5. A: Addition.
  6. A: Addition.

Using BODMAS, we solve expressions in this order:

  1. Brackets
  2. Division
  3. Multiplication
  4. Addition
  5. Subtraction

PEMDAS:

  1. Parenthesis ()
  2. Exponentiation **
  3. Multiplication *
  4. Division /
  5. Addition +
  6. Subtraction –

Multiplication and division have the same priority but always start from left to right.
Addition and subtraction have the same priority no matter to solve which the first but always go from left to right.

In our example, there are no brackets, so we start with division:
  1. First, solve 2 / 2, which is 1.
  2. Then, add 2 + 1, giving us 3.

So, a will be equal to 3.

If you want an addition to happen before division, you will use brackets like this: 
let a = (2 + 2) / 2;

Now, the result will be 2 because we do the addition inside the brackets first.

In summary, operator precedence is all about knowing the order in which operations are performed. Using BODMAS can help us solve complex calculations step by step.

Nullish coalescing operator

The nullish coalescing operator is written as two question marks ??.

As it treats null and undefined similarly, we’ll use a special term here, in this article. For brevity, we’ll say that a value is “defined” when it’s neither null nor undefined.

The result of a ?? b is:

  • if a is defined, then a,
  • if a isn’t defined, then b.

In other words, ?? returns the first argument if it’s not null/undefined. Otherwise, the second one.

The nullish coalescing operator isn’t anything completely new. It’s just a nice syntax to get the first “defined” value of the two.

We can rewrite result = a ?? b using the operators that we already know, like this: 
result = (a !== null && a !== undefined) ? a : b;

Now it should be absolutely clear what ?? does. Let’s see where it helps.
The common use case for ?? is to provide a default value.

For example, here we show user if its value isn’t null/undefined, otherwise Anonymous: 
let user;
alert(user ?? "Anonymous"); // Anonymous (user is undefined)
Here’s the example with user assigned to a name: 
let user = "Naseeb";
alert(user ?? "Anonymous"); //Naseeb (user is not null/undefined)

We can also use a sequence of ?? to select the first value from a list that isn’t null/undefined.

Let’s say we have a user’s data in variables firstName, lastName or nickName. All of them may be not defined, if the user decided not to fill in the corresponding values.

We’d like to display the user name using one of these variables, or show “Anonymous” if all of them are null/undefined.

Let’s use the ?? operator for that: 

let firstName = null;
let lastName = null;
let nickName = "Supercoder";

Shows the first defined value:

alert(firstName ?? lastName ?? nickName ?? "Anonymous"); // Supercoder

Here, the result is "Naseeb". The first value is false, so JavaScript checks the next value. Since "Naseeb" is a non-empty string (a "truthy" value), it returns "Naseeb".

Exercise

Let's solve a simple interview question! Imagine we have two variables with colors assigned to them:

let a = "Red";
let b = "Black";

Now, we want to swap the values of these variables so that:

  • a will store "Black"
  • b will store "Red"

To do this, we'll need a third temporary variable. This extra variable helps us hold onto the value of a while we switch the values around. Here’s how it works step-by-step:

1. Save the value of a into a new variable, c, so we don’t lose it:

let c = a; // c now holds "Red"

2. Set the value of a to b:

a = b; // a now holds "Black"

3. Assign the original value of a (stored in c) to b:

b = c; // b now holds "Red"
Now we’ve successfully swapped the values, and our variables look like this: 
a = "Black";
b = "Red";

This approach works because the temporary variable c acts as a "container" to hold one value while we swap the other two.
