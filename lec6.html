Objects With OOP

An object is a collection of key-value pairs. If we have properties that are highly related, we want to encapsulate them inside an object. For example, let’s say we’re building an application that draws different types of shapes, like circles, rectangles, and so on. We could declare multiple variables around circles.

In OOP object are used to Groups data and related functions together in one place. 

let radius = 1;
let x = 1;
let y = 1;

Here, we are defining multiple variables, and these variables are highly related, representing a circle. A better approach is to put these variables inside an object. This way, we can pass the object anywhere in the program or any function, making all these variables available within that object. In an object, we can include anything, such as other objects, Booleans, functions, etc. 

const circle = {
 radius: 1,
 x: 1,
 y: 1,
 location: {
 x: 1,
 y: 1
 },
 isVisible: true
};

In this example, we have an object (circle) with another object inside (location) and additional values. Objects are used to collect all related data in a single variable. But it’s not just about grouping related variables; we can also have functions that operate on these radius, x, and y variables. For example, we might have a function for drawing and another for moving a circle. These functions are highly related to the radius, x, and y variables, so it makes sense to include them inside the object.

With objects, we can access properties and functions anywhere in the program. Let's see how we can pass functions as properties of objects. The value of a property in an object can be anything. 

const circle = {
 radius: 1,
 location: {
 x: 1,
 y: 1
 },
 isVisible: true,
 draw: function() {
 console.log('draw');
 }
};

With this object, we don’t need variables outside of the object, like radius, x, and y. What we see here is called "object-oriented style" in programming, which is also referred to as Object-Oriented Programming (OOP). OOP is a programming style where we view a program as a collection of objects that communicate with each other. For example, we have a circle object with a few properties and a function.

In OOP terms, if a function is part of an object, we call that function a method. So, instead of saying circle.draw(); as a property, we can say it’s a method of the circle object. When calling a function within an object, we refer to it as method calling.

This is the difference between a function and a method: a method is simply a function that belongs to an object. This example is an easy way to create an object, but if our application gets more complex, we may use other ways to write objects. 

const dog = {
  name: 'Buddy',
 type: 'Dog',
 bark: function() {
 console.log('Woof! Woof!');
 },
 describe: function() {
 console.log('This is a ' + this.type + ' named ' + this.name);
 }
};

// Using the object
dog.bark(); // Output: 'Woof! Woof!'
dog.describe(); // Output: 'This is a Dog named Buddy'

Here, in an object the key itself is a string, but we sometimes wrap it with double or single quotes. However, most of the time, we write it directly without using string syntax.

MAP

In JavaScript, a Map is a collection of key-value pairs where each key can be of any type (unlike objects, where keys are limited to strings or symbols). You can use loops to iterate over the keys, values, or entries (key-value pairs) in a Map. Let me guide you step by step in simple terms.

1. Creating a Map

You can create a Map like this:

const myMap = new Map();
2. Adding Key-Value Pairs

Use the set() method to add key-value pairs to the Map:

myMap.set('name', 'Naseeb');

myMap.set('age', 23);

myMap.set('isStudent', false);
Now, myMap contains:
  • Key: 'name', Value: 'Naseeb'
  • Key: 'age', Value: 23
  • Key: 'isStudent', Value: false
3. Looping Through a Map

You can use loops to handle the values in a Map. Here are the common ways:

3.1. Loop through keys

Use the keys() method to get all the keys and loop through them: 

for (let key of myMap.keys()) {
 console.log(key); // Output: 'name', 'age', 'isStudent'
}
3.2 Loop through values

Use the values() method to get all the values and loop through them: 

for (let value of myMap.values()) {
 console.log(value); // Output: 'Naseeb', 23, false
}
3.3 Loop through key-value pairs (entries)

Use the entries() method to get both keys and values:

for (let [key, value] of myMap.entries()) {
  console.log(`Key: ${key}, Value: ${value}`);
}

Output:
Key: name, Value: Naseeb
Key: age, Value: 23
Key: isStudent, Value: false
3.4 Using forEach

You can also use the forEach method to loop through the Map: 

myMap.forEach((value, key) => {
  console.log(`Key: ${key}, Value: ${value}`);
});
4. Checking if a Key Exists

Use the has() method to check if a key exists in the Map: 

console.log(myMap.has('name')); // Output: true
console.log(myMap.has('address')); // Output: false
5. Getting a Value by Key

Use the get() method to retrieve the value for a specific key: 

console.log(myMap.get('age')); // Output: 23
6. Deleting a Key-Value Pair

Use the delete() method to remove a key-value pair: 

myMap.delete('isStudent');
console.log(myMap.has('isStudent')); // Output: false
7. Clearing the Map

Use the clear() method to remove all key-value pairs: 

myMap.clear();
console.log(myMap.size); // Output: 0
8. Getting the Size of the Map

Use the size property to get the number of key-value pairs: 

console.log(myMap.size); // Output: 3 (before clearing)

Example: Putting It All Together

Here’s a complete example: 

const myMap = new Map();
myMap.set('name', 'Naseeb');
myMap.set('age', 23);
myMap.set('isStudent', false);

// Loop through keys
for (let key of myMap.keys()) {
  console.log(key);
}

// Loop through values
for (let value of myMap.values()) {
  console.log(value);
}

// Loop through entries
for (let [key, value] of myMap.entries()) {
 console.log(`Key: ${key}, Value: ${value}`);
}

// Check if a key exists
console.log(myMap.has('name')); // true

// Get a value by key
console.log(myMap.get('age')); // 23

// Delete a key-value pair
myMap.delete('isStudent');

// Clear the map
myMap.clear();
console.log(myMap.size); // 0

Creating Objects Using Factory Functions

Introduction: We learned how to create an object using object literal syntax. However, in the previous code, there were some issues that need fixing. Imagine we want to create two circle objects. If we copy and paste the “object literal”, we’ll have code duplication, especially in the draw method. This approach is not ideal, especially if the “draw” method contains multiple lines or if the object has more methods. To avoid duplication, we use factory functions or constructor functions.

Original Object Literal Syntax Example 
const circle = {
 radius: 1,
 location: {
  x: 1,
  y: 1
 },
 isVisible: true,
 draw: function() {
  console.log('draw');
 }
};
const circle2 = {
 radius: 1,
 location: {
  x: 1,
  y: 1
 },
 isVisible: true,
 draw: function() {
  console.log('draw');
 }
};

In this code, creating a second circle object (circle2) results in duplicated code, especially the “draw” method. If we want to make our “draw” method more complex, duplicating code would be inefficient and hard to maintain. Instead, we can use a factory function to create circle objects dynamically.

Using Factory Functions

A factory function is a function that returns an object. It allows us to define the logic once and create multiple objects based on that logic.

Basic Factory Function Example: 
function createCircle() {
 return {
  radius: 1,
 location: {
  x: 1,
  y: 1
 },
 isVisible: true,
 draw: function() {
  console.log('draw');
 }
 };
}

This function returns a new circle object every time it is called. However, this code still hardcodes radius and location, making it less flexible. Let’s improve it by passing radius and location as parameters.

Adding Parameters to the Factory Function

By passing radius and location as parameters, we make our factory function flexible, allowing us to create circles with different properties: 

function createCircle(radius, location) {
 return {
    radius: radius,
    location: location,
    isVisible: true,
 draw: function() {
    console.log('draw');
 }
 };
}

Modern JavaScript Syntax

In modern JavaScript, if the key and value names are the same, we can simplify the code by omitting the key-value pairs and just writing the variable name: 

function createCircle(radius, location) {
 return {
    radius,
    location,
    isVisible: true,
 draw: function() {
    console.log('draw');
 }
 };
}
Additionally, we can simplify the “draw” method by using a shorthand syntax: 
function createCircle(radius, location) {
 return {
    radius,
    location,
    isVisible: true,
 draw() {
    console.log('draw');
 }
 };
}

Using the Factory Function

Now we can create multiple circle objects by calling “createCircle” with different values: 

const circle1 = createCircle(1, { x: 1, y: 1 });
console.log(circle1);

Output:
{
 radius: 1,
 location: { x: 1, y: 1 },
 isVisible: true,
 draw: [Function: draw]
}

The beauty of this factory function is that we defined the “draw” method only once. We can create as many circles as we want with different values, and they will all share the same “draw” method without code duplication.

Constructor function

Just like a factory function, a constructor function's job is to construct or create an object. The naming convention for constructors is different from factory functions. In factory functions, we use camel notation. Camel notation is when names like “createCircle” are written with the first word in lowercase and subsequent words capitalized (e.g., createCircle). This is called the camel case.

In contrast, there is another naming convention called Pascal notation. In Pascal notation, each word starts with an uppercase letter, like “CreateCircle”. When we want to create a constructor function, we should use Pascal case because other JavaScript developers expect this convention when reading code.

Remember the previous program using a factory function? In the constructor function, we use the “this” keyword. “this” is a keyword that references an object that is executing a piece of code. Imagine “this” as referencing an empty object. Using dot notation, we can access, read, and set properties of this object. In our new object, we want to add a property called radius and assign it a value from an argument passed to the parameter.

In JavaScript, objects are dynamic, so once we create them, we can add multiple properties and functions to them.

Example 
function Circle(radius) {
 this.radius = radius;
 this.draw = function() {
 console.log('draw');
 };
}
To create a circle object using this constructor function, we define: 
const circle = new Circle(1);

Using the new keyword, it creates a new JavaScript object, similar to const x = {};. We don’t need to return this; because this is done automatically behind the scenes. When we use the new keyword, three things happen:

  1. The “new” keyword creates an empty object.
  2. It then sets this in the function to point to this new object.
  3. It returns the new object automatically.

Factory Function Comparison:

 return {
    radius,
    draw() {
        console.log('draw');
 }
 };
}
To create an object using a factory function: 
const myCircle = createCircle(1);

In this case, we call the function to return an object.

Dynamic Nature of Objects

One important concept to understand about JavaScript is that it is dynamic. This means that once we create an object, we can always add new properties to it.

Example 
const circle = { radius: 1 };

Here, we have a single object circle with a single property, radius. Let's add a new property to it: 

circle.color = 'yellow';
console.log(circle);
Now, this object has two properties: radius and color.

Similarly, we can add a method to this object: 

circle.draw = function() {};

We can also delete existing properties and methods from an object using the “delete” keyword. For example:

delete circle.color;
delete circle.draw;

A lot of developers find it confusing to use the const keyword to define a constant object, as they think it makes the object completely unchangeable. However, as we can see, we have modified the circle object by adding and removing properties.

So, what type of "constant" is this? Declaring an object with const circle means we cannot reassign the variable circle itself. This variable remains constant in reference, meaning we cannot do something like:

circle = {};

Attempting to reassign the entire circle variable will result in an error. However, we can still add, change, and remove properties within the circle object itself.

Constructor Property

Understanding the Constructor Property in JavaScript
Every object in JavaScript has a property called constructor, which references the function used to construct or create that object. Let's explore this concept with two examples: 

// Factory function to create a circle object
function createCircle(radius) {
 return {
 radius,
 draw: function() {
 console.log('draw');
 }
 };
}
const circle = createCircle(1);

// Constructor function to create a circle object
function Circle(radius) {
    this.radius = radius;
    this.draw = function() {
    console.log('draw');
 };
}
const another = new Circle(1);

// Examining the constructor property
console.log(another.constructor);

// This returns the Circle function used to create the object

console.log(circle.constructor);

// This will return: function Object() { [native code] }
// This indicates that the object was created using the built-in Object constructor.

Explanation of Constructor Property

The constructor property of an object points to the function that was used to create it. When we call console.log(another.constructor), it returns the Circle function, showing that the another object was created using the Circle constructor.

When we call console.log(circle.constructor), it returns function Object() { [native code] }, which is a built-in constructor function. This is because circle was created using a factory function that returns an object literal. The JavaScript engine internally uses the Object constructor for object literals.

Creating Objects with Object Literal Syntax

Let's create an object using the object literal syntax: 

let x = {};

Internally, the JavaScript engine translates this to: 

let x = new Object();

This means that when we use object literal syntax, the JavaScript engine creates the object using the Object constructor function.

Other Constructor Functions in JavaScript

JavaScript provides a few other built-in constructors, such as:

  • new String();
  • new Boolean();
  • new Number();

However, using string literals (single quotes 'Naseeb', double quotes "Naseeb", and backticks `Naseeb`) is cleaner and more concise than using the String constructor.

For booleans, we use true and false directly instead of the Boolean constructor.

Value vs. Reference Types

In JavaScript, there are two categories of types: value types (also called primitives) and reference types. Understanding how these two categories behave is crucial before moving on to more advanced topics like prototypes.

Value (Primitive) Types

Value types or primitives include:

  • Number
  • String
  • Boolean
  • Undefined
  • Null
  • Symbol

Reference Types

Reference types include:

  • Object
  • Function
  • Array

In the previous topic, we learned that a function is an object, and the same goes for an array. In JavaScript, there are essentially primitives and objects.

Behavior of Primitives and Objects

Primitives and objects behave differently, and understanding this distinction is essential.

Example with Primitives: 
let x = 10;
let y = x;
x = 20;

In this example, x and y are two independent variables. Here, x becomes 20, and y remains 10. They are independent because when we work with primitives, the value (e.g., 10) is stored directly in the variable. When we copy x into y, the value stored in x is copied to y. Therefore, they are completely independent of each other.

Example with Reference Types 
let x = { value: 10 };
let y = x;
x.value = 20;

In this case, the values of both x and y will be 20. This is because when we use an object, the object itself is not stored in the variable. Instead, the object is stored somewhere else in memory, and the variable holds a reference (or address) to that location in memory. When we copy x into y, the reference is copied, not the actual object. Both x and y point to the same location in memory. Therefore, when we modify the object using either x or y, the changes are immediately visible in the other variable.

Example 2: 
let number = 10;
function increase(number) {
 number++;
}

increase(number);
console.log(number); // Output: 10

Here, the number did not increase when we called the increase function. This is because the number variable’s value is copied into the "number" parameter, which is local to the function. The number variable inside the function (number++) is completely independent of the original number variable that stores the value 10.

Inside a function it can’t take a value of external variables.

Within the function, we increment number by 1, so it becomes 11. However, after the function execution, this incremented value goes out of the function scope. When we log number to the console, we are dealing with the original number variable, not the one modified in the function.

Primitive values are copied by their value. Here, we are dealing with two independent copies, which is why we see 10 in the console.

Now, let’s change the number variable into an object. 
let obj = { value: 10 };
function increase(obj) {
 obj.value++;
}
increase(obj);
console.log(obj); // Output: 11

Explanation: In this case, when we call the increase function and pass obj, this object is passed by its reference. The local parameter inside the function points to the same object that we defined in the first line of the code.

Here, we are not dealing with two independent copies. Instead, we have two variables (the original obj and the parameter obj inside the function) that point to the same object. Any change we make to the object within the function is reflected in the original obj.

This is why the output of console.log(obj) shows 11.

Enumerating Properties of an Object

In previous topics, we learned how to use for...in and for...of loops to iterate over the properties of an object. Let’s master these concepts for a deeper understanding.

Example 
const circle = {
    radius: 1,
    draw() {
        console.log('draw');
 }
};

Using for...in Loop

To iterate over the properties of the circle object, we use the for...in loop: 

for (let key in circle) {
    console.log(key);
}

This loop will log the property names (radius, draw) to the console.

Accessing Property Values:

If we want to access the values of the properties, we use bracket notation: 

for (let key in circle) {
  console.log(key, circle[key]);
}

This will print both the property names and their corresponding values.

Using for...of Loop:

Attempting to use the for...of loop directly on an object will result in an error: 

for (let key of circle) {
 console.log(key);
}

// Error: circle is not iterable

Explanation: For...of loop can only be used with iterables such as arrays, strings, and maps. Since an object is not inherently iterable, the for...of loop cannot be used directly with it.

Using Object.keys() Method:

To make an object iterable, we can use the Object.keys() method, which returns an array of the object's keys. Since arrays are iterable, we can then use the for...of loop: 

for (let key of Object.keys(circle)) {
 console.log(key);
}

This will output the property names as an array.

Background on Object Constructor:

Object is an inbuilt constructor function in JavaScript. When we create an object using the literal syntax, such as: 

const x = { value: 1 };
Internally, JavaScript calls Object(): 
const x = new Object();
Using Object.entries() Method:

The Object.entries() method returns an array of the object's own enumerable property [key, value] pairs. We can iterate over these entries using the for...of loop: 

for (let entry of Object.entries(circle)) {
 console.log(entry);
}

This will output each key-value pair as an array (e.g., ['radius', 1], ['draw', function]).

Using the in Operator:

To check if a specific property exists in an object, we use the in operator: 

if ('radius' in circle) {
 console.log('yes');
}

This checks if the circle object has the radius property and logs ‘yes’ if true.

Cloning an Object

In the last topic, we learned how to enumerate the properties of an object. Using this technique, we can copy all the properties of one object into another object.

Example 1: Copying an Object Using a For-In Loop

Consider the following circle object:

const circle = {
    radius: 1,
    draw() {
      console.log('draw');
 }
};

Now, let's create another circle object that is a copy of the original circle object. Initially, we start with an empty object: 

const another = {};

We are going to use a for-in loop to iterate all the properties of the circle object and copy them into the “another” object: 

for (let key in circle) {
    another[key] = circle[key]; // Use bracket notation to access the key
}
Here, the left side (another[key]) is assigned the value of the right side (circle[key]). For example: 
// another['radius'] = circle['radius'];

In this line, we are reading the radius property from the circle object and assigning that value to the “another“object. If we print the another object with console.log(another);, we will get the complete clone of circle object

This method of copying/cloning an object is a bit old. In modern JavaScript, we have simpler ways to achieve the same result.

Example 2: Copying an Object Using Object.assign()

A more modern way to clone an object is to use the Object.assign() method: 

  const another = Object.assign({}, circle);

In this example, an empty object ({}) is created, and the properties of the circle object are copied into it and assigned to the “another” variable. We can also add additional properties as needed: 

const another = Object.assign({ color: 'yellow' }, circle);
console.log(another);

here the color: ‘yellow’ property and the circle object is assigned to “another” object.

Example 3: Copying an Object Using the Spread Operator

The simplest way to clone objects in modern JavaScript is by using the spread operator (...): 

const another = { ...circle };

This operator copies all the properties and methods of the circle object and places them inside the curly braces, assigning them to the “another” variable. We will get the same result as before.

Garbage Collection

In low-level languages like C/C++, when creating an object, we allocate memory to it. When we are done, we need to deallocate the memory. In JavaScript, we don't have this concept. We can easily create an object, and at the time we initialize this object, memory is automatically allocated to it. 

let circle = {};
console.log(circle);

We can use objects, and when we are done, we don't need to deallocate the memory. In the JavaScript engine, there is a feature called the garbage collector. This garbage collector is used to find variables or constants that are no longer used and then deallocate the memory that was allocated to them earlier. As JavaScript developers, we do not need to worry about memory allocation and deallocation; it is done automatically behind the scenes.

We have no control over when the garbage collector runs or what variables it removes from memory. In complex algorithms, the garbage collector runs in the background, figures out which variables are no longer in use, and automatically deallocates their memory.
