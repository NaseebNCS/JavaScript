Escape Notation

Escape notations are special characters. If we want to use them, we need to encode them using escape sequences. For example, if you have a single quote in your string, the JavaScript engine might get confused and think it marks the end of the string. 

Incorrect code:
const message = 'This is my 'first message';

Here, the JavaScript engine gets confused. It thinks the second single quote ends the string.

To fix this, we need to prefix the single quote inside the string with a backslash (\), which is the escape character.

Corrected code using escape notation:
const message = 'This is my \'first message';

Now, \' is an escape sequence that allows a single quote to be part of the string.

console.log(message); // Output: This is my 'first message'

Another escape sequence is \n, which represents a new line.

Example with \n:

const multilineMessage = 'This is line one.\nThis is line two.';
console.log(multilineMessage);

Output:
// This is line one.
// This is line two.

A useful method you can use with strings is .split(). If you apply it to a string, it converts the string into an array. Each word or segment is separated into a different element of the array based on the delimiter you specify

Example with .split():

const words = message.split(' ');
console.log(words);

// Output: ['This', 'is', 'my', "'first", "message'"]

Template Literals:

We have learned to use \n for inserting a new line in a string. However, I don't like this approach because it doesn't allow us to visualize what the output will look like when we print the string. The output and the code won't look similar. 

const message = 'This is my\nfirst message';

Output
// This is my
// first message

But both the code and output are different.

What if we want the output and the code to look the same? We can concatenate the string using the + operator and use the Enter key to align the content in a new line. We can also shift the first line to the second line using Enter. This approach will look somewhat like the output.

However, we still have the backslash \n, which creates some visual noise in the code. If we want the first word in a string to be in single quotes, we will need to use a backslash \'.

To make our code cleaner and easier to read, we use template literals.

What are Template Literals?

In JavaScript ES6, template literals are introduced using backticks (`). These make our code cleaner and easier to visualize.

Example of a Template Literal: 

const anotherMsg = `This is my first message`;

Where we need a new line, we can simply press Enter:

const anotherMsg = `This is my
                    first message`;

If we want to surround the first word in single quotes, we can write: 

const anotherMsg = `'This' is my first message`;

No need to use backslashes (\') or the + operator.

Use Case: Writing Email Templates

This is particularly useful when we want to send email messages in our applications. Here's an example of writing an email template using template literals: 

const email = `Hi Lucky,
                    Thank you for joining my mailing list.
              Regards,
                    Naseeb Khan`;

There is no need to use + or backslash characters.

Placeholders in Template Literals

Another benefit of template literals is the ability to use placeholders inside a string.

If we want to include dynamic names in the email, we can use variables: 

const receiver = "Lucky";

const sender = "Naseeb Khan";

const email = `Hi ${receiver},
Thank you for joining my mailing list.
Regards,
${sender}`;

${} are placeholders for variables or expressions. We can also add any type of expression, such as: 

const sum = `The result of 2 + 2 is ${2 + 2}`;

We can even use functions within template literals: 

const greet = (name) => `Hello, ${name}!`;
const message = greet('Lucky'); // Output: "Hello, Lucky!"

Finding Elements (Primitives):

Finding elements depends on whether you are storing primitives or references in an array. Let's start with primitives. We have an array for testing:

const numbers = [1, 2, 3, 4];

To find elements in an array, there is the .indexOf() method, which returns the index of the element if it exists in the array. If not, it returns -1. 

console.log(numbers.indexOf(2)); // Output: 1

Similar to .indexOf(), there is another method called .lastIndexOf(). It returns the last index of the given element if it exists. If not, it also returns -1 

const searchLastNumber = [1, 2, 3, 4, 1];
console.log(searchLastNumber.lastIndexOf(1)); // Output: 4

If we want the result to be true or false, we can write a program like this: 

const result = numbers.indexOf(1) !== -1;
console.log(result); // Output: true

This method is considered old. We now have a newer and simpler way to achieve the same result using the .includes() method: 

console.log(numbers.includes(1)); // Output: true

All these methods have an optional second parameter, which specifies the starting index for the search. 

console.log(numbers.indexOf(1, 2)); // Output: -1

Here, we are searching for 1 starting from index 2. The second parameter is the location where the search begins.

Finding Elements (Objects)

Finding primitives is different from finding reference (or object) types. Let's see how: 

const courses = [
    { id: 1, name: 'Naseeb' },
    { id: 2, name: 'Lucky' }
];

Let's try to find the course with the name 'Lucky' in this array. The includes method we learned earlier won't work here.

const result = courses.includes({ id: 2, name: 'Lucky' });
console.log(result); // Output: false

This happens because the object in the array is different from the object we are searching for using the includes() method. These are two different objects with two different references in memory.

Objects are reference types, and when we check for equality, their references are compared. In this case, they have two different references, which is why we get false.

Finding Objects Using find Method:

If we have an array of reference types (like objects), we can use the find method. The find method takes a function as an argument. This function is called a predicate or callback function.

The predicate function is used to determine if a given element exists in the array. It takes a parameter (representing an element in the array) and should return a Boolean value (true or false). 

const course = courses.find(function(course) {
 return course.name === 'Lucky';
});

console.log(course);
// Output: { id: 2, name: 'Lucky' }

If the element exists in the array, the find method returns the complete element. If it doesn’t exist, it returns undefined.

Finding the Index Using findIndex Method:

Similarly, there is another method called findIndex. This method returns the index of the searched element. If the element isn’t found, it returns -1. 

const index = courses.findIndex(function(course) {
 return course.name === 'Lucky';
});

console.log(index);
// Output: 1

// If the element is not found in the array:
const index = courses.findIndex(function(course) {
 return course.name === 'a';
});

console.log(index);
// Output: -1

Arrow Functions:

Regular Function

function addNumbers(a, b) {
 return
}

What it does:

  • It takes two numbers as inputs (a and b).
  • It returns their sum (a + b).

Arrow Function

  • const addNumbers = (a, b) => { return a + b; }
  • const addNumbers = (a, b) => a + b;

Explanation of changes:

  1. Function name: Instead of using the function keyword, we assign the function to a variable called addNumbers.
  2. Arrow syntax: The => replaces the function keyword and tells JavaScript this is an arrow function.
  3. Return keyword: For one-line functions, you don't need to explicitly write return or use curly braces ({}). The expression after the arrow is automatically returned.

Previous Example:

In the last lecture, we learned about a predicate function, which is passed to the find method. 

const courses = [
 { id: 1, name: "Naseeb" },
 { id: 2, name: "Khan" },
];

const course = courses.find(function (course) {
 return course.name === "Naseeb";
});

In ES6, there is a shorter and clearer way to write the same code using arrow functions. Whenever you want to pass a function as a callback or as an argument to another method, you can use the arrow function syntax.

In an arrow function:

  • The function keyword is removed.
  • The parameters are separated from the function body using a fat arrow (=>).
  • If there is a single parameter, you can write it without parentheses.
  • If there are no parameters, you need to use empty parentheses (()).
  • If the function contains a single line of code and returns a value, you can make the code shorter by removing the return keyword and curly braces, placing the expression directly after the arrow.

Example using arrow function: 

const course = courses.find(course => course.name === "Naseeb");

Here, we say course "goes to" course.name === "Naseeb". Finally, we are finding the course with the name equal to "Naseeb".

Adding Elements

We can add elements to an array, but we can't reassign an array declared with the const keyword. 

const numbers = [3, 4, 5];

If we try to reassign this array, we will get an error.

numbers = []; // This will throw an error.

However, it is perfectly fine to modify the content of this array. Arrays are objects in JavaScript. By using dot notation, we can access all the properties and methods defined for arrays. 

numbers.push(6, 7, 8, 9); // Adds elements to the end of the array.
numbers.unshift(0, 1, 2); // Adds elements to the beginning of the array.
numbers.splice(2, 0, 3, 4); // Adds elements to the middle of the array.

Explanation of Methods:

Adding elements at the end of an array: Use the .push() method. It allows you to pass one or more arguments, which will be placed at the end of the array. 

numbers.push(6, 7, 8, 9);

Adding elements at the beginning of an array: Use the .unshift() method. It pushes all existing elements to the right and adds new elements to the left of the array. 

numbers.unshift(0, 1, 2);

Adding elements to the middle of an array: Use the .splice() method. This method requires three arguments:

  • First: The position where you want to add or delete elements.
  • Second: The number of elements to delete (if any).
  • Third: The new elements to add at the specified position.
numbers.splice(2, 0, 3, 4); // Adds 3 and 4 at index 2.

Removing Array Elements:

To remove elements from an array, there are three common methods. In the previous topic, we learned three ways of adding elements to an array; similarly, here we have three ways to remove elements from an array

Removing the last element: .pop() The .pop() method removes the last element from an array and returns it. 

const numbers = [1, 2, 3, 4];
const last = numbers.pop();

console.log(numbers); // Output: [1, 2, 3]
console.log(last); // Output: 4

Removing the first element: .shift() If we want to remove the first element from an array, we use the .shift() method. Similar to .pop(), it removes an element from the array and returns it. 

const numbers = [1, 2, 3, 4];
const first = numbers.shift();
console.log(numbers); // Output: [2, 3, 4]
console.log(first); // Output: 1

Removing an element from the middle: .splice() To remove an element from the middle of an array, we use the .splice() method.

  • The first parameter specifies the starting index for removal.
  • The second parameter specifies the number of elements to remove.
const numbers = [1, 2, 3, 4, 5];
numbers.splice(2, 1); // Removes 1 element starting at index 2
console.log(numbers); // Output: [1, 2, 4, 5]

If we want to delete multiple elements from an array, we can adjust the second parameter. 

const numbers = [1, 2, 3, 4, 5];
numbers.splice(2, 3); // Removes 3 elements starting at index 2
console.log(numbers); // Output: [1, 2]
numbers.splice(2); // Remove all elements after index 2

Emptying an Array

In the previous lesson, we learned how to remove an element from an array. But what about emptying an entire array? How do we remove all the elements from an array? Let's explore several solutions:

Solution 1: Reassigning to an Empty Array

et numbers = [1, 2, 3, 4];
numbers = [];

Now the numbers array is empty. However, note that the original array still exists in memory. If there are no other references to this array, it will eventually be removed by the garbage collector.

If another variable is referencing the same array, the original array will not be garbage collected. 

let numbers = [1, 2, 3, 4];
let numbers1 = numbers;
numbers = [];

Here, numbers array is reassigned to a new empty array. However, numbers1 is still pointing to the original array. This solution works well when there are no other references to the original array.

Solution 2: Setting length to 0

let numbers = [1, 2, 3, 4];
numbers.length = 0;

This will empty the array, and any other variables referencing the original array will also reflect the change (i.e., they will also appear empty). It doesn’t matter how many references exist; all of them will lead to an empty array.

Solution 3: Using splice()

let numbers = [1, 2, 3, 4];
numbers.splice(0, numbers.length);

Here:

  • 0 indicates where to start removing elements.
  • numbers.length specifies how many elements to remove.

After this, the numbers array will be empty.

Solution 4: Using pop() in a Loop

let numbers = [1, 2, 3, 4];

while (numbers.length > 0) {
 numbers.pop();
}

The pop() method removes elements from the end of the array one at a time. The loop continues until the array is empty.

Note: This method is not recommended for large arrays due to its performance cost.

Which one to use:

  • Use Solution 1 if you are okay with reassigning the array and there are no other references.
  • Use Solution 2 if you want to empty all references pointing to the original array.
  • Use Solution 3 for flexibility with partial or complete array removal.
  • Avoid Solution 4 for performance reasons, especially with large arrays.

Combining and Slicing Arrays

Combining Arrays

In JavaScript, arrays can be combined using the concat() method. This method does not modify the original arrays but returns a new array that is the combination of the input arrays. 

const first = [1, 2, 3];
const second = [4, 5, 6];

// Combining both arrays into a new array
const combined = first.concat(second);
console.log(combined); // Output: [1, 2, 3, 4, 5, 6]

The concat() method creates a new array by combining the input arrays. The original arrays remain unchanged.

Slicing Arrays

The slice() method is used to extract a portion of an array into a new array. This method takes two optional parameters:

  1. Start Index - The index where slicing begins.
  2. End Index - The index where slicing ends (not inclusive).

1. Slicing a Specific Range: 

const sliced = combined.slice(2, 4);
console.log(sliced); // Output: [3, 4]

This starts slicing from index 2 and ends before index 4.

2. Slicing from an Index to the End: 

const sliced = combined.slice(2);
console.log(sliced); // Output: [3, 4, 5, 6]

This starts slicing from index 2 and continues to the end of the array.

3. Creating a Copy of the Array: 

const copy = combined.slice();
console.log(copy); // Output: [1, 2, 3, 4, 5, 6]

If no parameters are passed, slice() creates a copy of the entire array.

Understanding Primitive and Reference Types

When working with concat() and slice(), it’s important to understand how primitive and reference types behave:

1. Primitive Types:

  • If the array contains primitive values (e.g., numbers, strings), these are copied by value to the new array.

2. Reference Types:

If the array contains objects, the objects themselves are not copied. Instead, their references are copied. This means the original and new arrays share the same objects. 

const first = [{ id: 1 }];
const second = first.concat([]);
first[0].id = 10;

The change in the original array reflects in the combined array because objects are copied by reference. 

console.log(first); // Output: [{ id: 10 }]
console.log(second); // Output: [{ id: 10 }]
The same principle applies to the slice() method:
  • Primitive types are copied by value.
  • Reference types are copied by reference.

Recap

  • Use concat() to combine arrays into a new array
  • Use slice() to extract parts of an array or create a copy.
  • Understand the difference between copying primitive and reference types to avoid unintended side effects.

The Spread Operator

In the last topic, we learned how to use the “.concat()” method to combine arrays. However, there is another way to achieve the same result using the spread operator in ES6. Let's explore why this approach is cleaner. 

const first = [1, 2, 3];
const second = [4, 5, 6];

Previous Method:

Using the “.concat()” method: 

const combined = first.concat(second);

Spread Operator:

Using the spread operator, the same result can be achieved as: 

const combined = [...first, ...second];

Here, we have the first array followed by the second array.

Why Use the Spread Operator?

With the spread operator, we have more flexibility. For example, if we want to add elements between the arrays, at the start, or at the end, we can do so easily: 

const combined = [...first, 'a', ...second, 'b'];

If we wanted to achieve the same thing using the .concat() method, it would be more complex.

Copying an Array

The “.slice()” method, when used without parameters, returns a copy of an array.

Old Method with “.slice()”: 

const copy = combined.slice(); // Returns a copy of the combined array.

Using the spread operator in ES6, the same result can be achieved more concisely: 

const copy = [...combined]; // Returns a copy of the combined array.

Conclusion

The spread operator provides a cleaner and more flexible approach for array operations such as combining or copying arrays in modern JavaScript.

Iterating an Array

We have learned various loops in JavaScript. If we want to iterate over an array, we can use the for...of loop. 

const numbers = [1, 2, 3];
for (let number of numbers) {
 console.log(number); // Output: 1, 2, 3
}

For each iteration, the number variable is set to one of the elements in the numbers array. Similarly, there is another way to iterate over an array, commonly seen in tutorials, which uses the forEach method. All arrays have the .forEach() method, which takes a callback function as a parameter. This callback function also takes an argument, in this case, number. 

numbers.forEach(function(number) {
 console.log(number);
});

When we call the .forEach() method, the callback function passed to it is executed for each element in the array. Each element is passed as an argument to this function. We can simplify this code by using the arrow function syntax. This allows us to remove the function keyword and use a fat arrow (=>) between the parameter and the body of the function. 

numbers.forEach((number) => {
 console.log(number);
});

If there is a single parameter, we don’t need to use parentheses around the parameter: 

numbers.forEach(number => console.log(number); );

The callback function passed to the forEach method can also optionally take a second parameter, which is the index of each element in the array. This allows us to print the index along with the element. 

numbers.forEach((number, index) => {
 console.log(`Index: ${index}, Value: ${number}`);
});

Join & Split Arrays

Another useful method of arrays is the join() method.
We want to join the elements of an array, and that's where the join() method comes in handy. When we use .join() or any other method, we might notice a question mark (?) in the guidelines provided by the code editor. This question mark indicates that the parameter is optional.

The type of the join() parameter is a string. We can optionally pass a string to the method, which will act as a separator between the elements of the array. It's important to note that the join() method returns a string. 

const numbers = [1, 2, 3];
const joined = numbers.join(',');
console.log(joined);
// Output: "1,2,3"

The split() Method

Another handy method, similar to join(), is the split() method. However, the split() method is not a part of arrays but is instead a method of strings. 

const message = "Subscribe To NaseebCodeStudio";
const parts = message.split(' ');
console.log(parts);
// Output: ["Subscribe", "To", "NaseebCodeStudio"]

The split() method breaks the original string into individual words or characters, depending on the delimiter passed to it, and returns an array.

Joining the Split Parts

We can use the join() method on the resulting array to combine the parts again using a different separator. 

const combined = parts.join('_');
console.log(combined);
// Output: "Subscribe_To_NaseebCodeStudio"

This technique is particularly helpful when creating an URL slug.

Example of URL Slug Creation

Let's consider a search query on Google, such as:

Query:

what is JavaScript?

Google converts the spaces in this query into different types of slugs. For example: https://www.google.com/search?q=what+is+javascript&rlz

In the URL slug, all the white spaces are replaced with +.

Creating a Custom URL Slug

To convert a string into a URL-friendly slug, we can use split() to break the string and join() to combine it with a new separator: 

const query = "what is javascript";
const queryParts = query.split(' ');
const urlSlug = queryParts.join('+');
console.log(urlSlug);

// Output: "what+is+javascript"

Recap

To convert strings into URL slugs or manipulate them effectively:

  1. Split the string using split().
  2. Join the parts using join() with the desired separator.

This method ensures flexible manipulation of strings and arrays in JavaScript.

These methods are very easy to use with numbers and strings. However, when we have objects in an array, they don't work by default. We need to do some extra work.

Example with Objects:

const courses = [
    { id: 3, name: 'ReactJS' },
    { id: 1, name: 'Node.js' },
    { id: 2, name: 'JavaScript' }
];
  • Attempting to sort without a comparison function won't work as expected
  • Sorting requires a comparison function to determine the order.

The .sort() method optionally takes a comparison function. This function compares two items (a and b) and determines their order. If a should appear before b, it returns -1; if a should appear after b, it returns 1; if they are equal, it returns 0.

Example of Sorting with a Comparison Function:

courses.sort(function (a, b) {
 if (a.name < b.name) return -1;
 if (a.name > b.name) return 1;
 return 0;
});

console.log(courses);

Output:

    { id: 1, name: 'Node.js' },
    { id: 3, name: 'ReactJS' }
]

However, this sorting is case-sensitive. To exclude case sensitivity, we can convert the name properties to the same case using .toLowerCase() or .toUpperCase()

Example of Case-Insensitive Sorting:

courses.sort(function (a, b) {
 const nameA = a.name.toLowerCase(); // Convert to lowercase
 const nameB = b.name.toLowerCase(); // Convert to lowercase

 if (nameA < nameB) return -1;
 
 if (nameA > nameB) return 1;
   return 0;
});

console.log(courses);

Output (case-insensitive):
[
    { id: 2, name: 'JavaScript' },
    { id: 1, name: 'Node.js' },
    { id: 3, name: 'ReactJS' }
]

Now, our sort algorithm will work properly, ignoring case sensitivity.

Note:

If you want to understand how the comparison works with upper-case and lower-case letters, refer to the ASCII table. In ASCII, uppercase letters have lower values than lowercase letters (A-Z comes before a-z), which is why converting both to the same case ensures consistent sorting.

Testing the Elements of an Array

There are two new methods in JavaScript called every and some. These methods are used to check whether there are positive values in an array or not. They are very useful methods.

1. every Method:

In every method, we pass a callback function. IntelliSense shows that this callback function can take three parameters:

  • The first parameter is the value, which represents the current element in the array.
  • The second parameter is the index, which is the index of the current element in the array.
  • The third parameter is the array itself.

While using this function, we don’t need to pass all the parameters. We can use only the ones we need.

In the example below, we are going to check whether all the values in the array are positive numbers: 

const numbers = [1, 2, 3];
const allPositive = numbers.every(function (value) {
    return value >= 0;
});

console.log(allPositive); // true

How it works:

When this function runs, it will start checking each value in the array. As soon as it finds a value that does not match the criteria (in this case, value >= 0), it will stop searching and return false.

2. some Method:

Some method works similarly but has a different purpose. It checks whether at least one element in the array matches the criteria. 

const numbers = [1, -1, 2, 3];

const atLeastOnePositive = numbers.some(function (value) {
 return value >= 0;
});

console.log(atLeastOnePositive); // true

How it works:

As soon as the function finds a number that matches the criteria (in this case, value >= 0), it will terminate and return true. It doesn’t matter what values come after the first matching element in the array.

Recap:

every Method:

  • Checks if all elements in an array meet a specified condition.
  • Stops as soon as it finds an element that doesn't match the condition.
const numbers = [1, 2, 3];
const allPositive = numbers.every(value => value >= 0);
console.log(allPositive); // true

some Method:

  • Checks if at least one element in an array meets a specified condition.
  • Stops as soon as it finds an element that matches the condition. 
const numbers = [1, -1, 2, 3];
const atLeastOnePositive = numbers.some(value => value >= 0);
console.log(atLeastOnePositive); // true

Key Difference:

  • every ensures all elements satisfy the condition.
  • some checks for at least one matching element.

Both methods are efficient because they stop processing as soon as the condition is met (or not).

Filtering an Array

To filter an array based on certain criteria, let's say we only need the positive numbers from an array. We are going to use the filter method. When we call the filter method, it iterates over the array and executes a callback function for each number/element. If the element meets the criteria, it will be added to a new array.

How to Use:

Here, we need to pass a callback function to the filter method. This function has three parameters: value, index, and array. Depending on our requirements, we can use these parameters. In this example, we just want to check if the number is positive 

const numbers = [1, -1, 2, 3];

// Using a regular function
const filtered = numbers.filter(function(value) {
 return value >= 0;
});
console.log(filtered); // Output: [1, 2, 3]

// Using an arrow function
const filteredArrow = numbers.filter(n => n >= 0);

Both approaches are valid, but using an arrow function is considered good practice because it is more concise and readable.

Mapping an Array

Another very useful method in JavaScript is the map method. Using this method, we can map each item in an array to something else. In the last topic, we had an array that was filtered using the filter function. Now, let’s say we want to construct some HTML markup using the elements in that array. Here, we are mapping each element into a bullet point. To achieve this, we use the <ul> or <ol> tags with <li> elements. or if we want to create another array based on old array we use map. 

const numbers = [1, -1, 2, 3]; // Original array
const filtered = numbers.filter(n => n >= 0); // Filter positive numbers
const items = filtered.map(n => '<li>' + n + '</li>'); // Map each number to an <li> element
console.log(items);

At this point, we have an array of strings. We can use the join method to combine the elements of this array into a single string. 

const html = items.join(); // Joins the array into a single string
console.log(html);

Now, instead of an array, we have a string.
Note: By default, the join method uses a comma as a separator.
However, we don’t want commas in our HTML markup. Let’s combine the elements with an empty string as a separator: 

const html = items.join(''); // Join with no separator

The only thing remaining is wrapping this in a <ul> element. Here’s the simple approach: 

const html = '<ul>' + items.join('') + '</ul>'; // Wrap in <ul> tags

Destructuring Array

Destructuring means taking specific values from an array and storing them in variables. It is like picking selected items from an array, not filtering or changing the original array. 

let arr = [1, 2, 3, 4];
let [a, b] = arr;

console.log(a, b); // 1 2

let [x, y, , z] = arr;

Mapping Objects:

We can also use the map method to map an array into objects. 

const numbers = [1, 2, 3]; // Original array
const filtered = numbers.filter(n => n >= 0); // Filter positive numbers
const items = filtered.map(n => {
 const obj = { value: n }; // Create an object for each element
 return obj;
});

Technically, we don’t need to declare the object as a separate constant. We can simplify this: 

const items = filtered.map(n => {
 return { value: n }; // Directly return the object
});

We can simplify further since the body of the arrow function has only a single line of code: 

const items = filtered.map(n => { value: n });

However, this will not work as expected because the curly braces are treated as a code block instead of an object. To fix this, we need to wrap the object in parentheses to ensure it is returned as an object: 

const items = filtered.map(n => ({ value: n })); // Correct syntax

Key Notes:

  1. Both the map and filter methods return new arrays; they do not modify the original array.
  2. These methods are chainable, meaning you can call one method after another.

Example of Chaining:

const numbers = [1, -1, 2, 3];
const items = numbers
 .filter(n => n >= 0) // Filter positive numbers
 .map(n => ({ value: n })); // Map to objects

In this example, the filtered array is used only once, so we don’t need to explicitly store the intermediate result in a constant. Here’s the same logic written with chaining: 

const items = numbers
 .filter(n => n >= 0)
 .map(n => ({ value: n })); // Chained methods

When chaining multiple methods, it’s good practice to write each method call on a new line. This makes the code cleaner and more readable: 

const items = numbers
 .filter(n => n >= 0) // Filter positive numbers
 .map(n => ({ value: n })) // Map to objects
 .filter(obj => obj.value > 1) // Filter objects with a value > 1
 .map(obj => obj.value); // Extract the `value` propert

Explanation:

  1. filter(n => n >= 0): Filters the array to include only positive numbers.
  2. map(n => ({ value: n })): Maps each number to an object with a value property.
  3. filter(obj => obj.value > 1): Filters objects to include only those with a value greater than 1.
  4. map(obj => obj.value): Maps the filtered objects back into an array of numbers.

This demonstrates how filter and map can be combined to process data in a clean, functional style.

Reducing an Array

Once again, we have an array with numbers, and we want to calculate the sum of all the numbers in this array. This is like calculating the total price of items in a shopping cart, where each number represents the price of an item in the cart.

Here is an algorithm to calculate the sum of all the numbers in the array:

  1. Start with a variable called sum, initialized to 0.
  2. Loop over the array, get each number (or element), and add it to sum.
const numbers = [1, 2, 3, 4];

let sum = 0;
for (let n of numbers) {
  sum += 
}

console.log(sum); // Output: 10

There is a cleaner and more elegant way to write the same code using the reduce method. All arrays in JavaScript have a reduce method. With the reduce method, we can reduce all the elements in an array into a single value. That single value can be anything, such as a sum, product, or concatenated string.

The reduce method takes a callback function with two parameters:

  1. accumulator: A variable that stores the accumulated result.
  2. currentValue: A variable that holds the current element being processed in the array.

The currentValue represents the current element of the array, and we add it to the accumulator. Simply put, we return accumulator + currentValue in the callback function.

Lastly, we initialize the accumulator to 0. 

const sum = numbers.reduce((accumulator, currentValue) => {
 return accumulator + currentValue;
}, 0);

console.log(sum); // Output: 10

If we don’t initialize the accumulator (for example, by omitting 0 as the second argument to the reduce method), it will, by default, be initialized to the first element of the array.

How this program works:

  • The reduce method iterates through the array.
  • The accumulator starts with the value you provide (or the first element if no initial value is provided).
  • For each element in the array, the currentValue is added to the accumulator.
  • This process continues until all elements in the array are processed.
  • The final value of the accumulator is returned.

This approach is concise and preferred in many cases when working with arrays.
