What can we do with JavaScript?

At first, JavaScript was only used in browsers to make web pages interactive. But thanks to big support from a large community and companies like Facebook and Google,
we can now use JavaScript to build entire websites, apps for phones, real-time chat apps, video streaming services, command-line tools, games, and more.

Where does JavaScript code run?

JavaScript was originally created to work only in web browsers, meaning it could only run inside them. Every browser has its own JavaScript engine, which is a tool that helps execute JavaScript code. For example, the Firefox browser uses the "Spider Monkey" engine, while Chrome uses the "V8" engine.

Node

In 2009, a smart engineer named Ryan Dahl took the V8 JavaScript engine (the one used in Chrome) and combined it with a C++ program. He named this program "Node." Basically, Node is a C++ program that includes Google's V8 engine. The cool part is that with Node, JavaScript code can now run outside of a browser. Instead of needing a browser to run the code, we can now use Node to execute JavaScript.

This means that JavaScript can be used to build the backend (server side) of web and mobile applications, not just for front-end work in browsers. Both the browser and Node provide a runtime environment where JavaScript code can run.

Compiler

In programming, when we choose C++, we write C++ code in a file with the .cpp extension. This file contains all the instructions we want to give the computer. However, computers don't understand C++ directly. To run the program, we need a compiler.

The compiler converts the C++ code into machine language, which is a binary language made up of 0s and 1s. This binary code is stored in a file with a .exe extension, also called an executable file. This file is created by compiler. This .exe file is what the computer can understand and execute. Finally, we run this .exe file to make the computer perform the tasks we’ve programmed,

But in JS we use browsers’ engines to execute, Engines are: V8 Engine, SpiderMonkey, JavaScriptCore (also known as Nitro), Chakra, Hermes etc.

Normal Language VS OOPS VS DSA?

The differences between normal language, object-oriented programming (OOP), and data structures and algorithms (DSA):

Normal Language

  • Normal languages, such as English, Spanish, or French, are used for communication between humans
  • They follow grammar rules, syntax, and semantics specific to each language.
  • Normal languages are not designed for programming or solving computational problems

Object-Oriented Programming (OOP):

  • Object-oriented programming is a programming paradigm based on the concept of "objects," which can contain data (attributes) and code (methods).
  • It focuses on organizing code into reusable objects, allowing for modular and scalable software development.
  • OOP principles include encapsulation, inheritance, and polymorphism, which help in building complex software systems.
  • Examples of OOP languages include Java, C++, Python, and Ruby

Data Structures and Algorithms (DSA):

  • Data structures are ways of organizing and storing data in a computer's memory, while algorithms are step-by-step procedures or formulas for solving specific problems.
  • DSA focuses on understanding how to efficiently store and manipulate data and how to design and analyze algorithms to solve computational problems.
  • Common data structures include arrays, linked lists, stacks, queues, trees, and graphs, while algorithms include sorting, searching, graph traversal, and dynamic programming.
  • DSA is essential for writing efficient and scalable software and is a fundamental topic in computer science and software engineering.

Differences Between IDE, interpreter, code editor, and compiler?

The differences between IDE, interpreter, code editor, and compiler:

  1. IDE (Integrated Development Environment):
    • An IDE is a software application that provides comprehensive tools and features for software development.
    • It typically includes a code editor, debugger, compiler or interpreter, build automation tools, and other features to streamline the development process.
    • Examples include Visual Studio, IntelliJ IDEA, and Eclipse.
  2. Interpreter:
    • An interpreter is a program that directly executes code written in a high-level programming language without the need for compilation
    • It reads the source code line by line, translates it into machine code or intermediate code, and executes it immediately.
    • Interpreters are commonly used for scripting languages like Python, JavaScript, and Ruby
  3. Code Editor:
    • A code editor is a simple software tool used for writing and editing source code.
    • It provides features like syntax highlighting, code formatting, auto-completion, and basic debugging capabilities.
    • Code editors are lightweight and focused on editing code, unlike IDEs which offer a broader set of development tools.
    • Examples include Sublime Text, Visual Studio Code, and Atom.
  4. Compiler:
    • A compiler is a program that translates source code written in a high-level programming language into machine code or bytecode.
    • It processes the entire source code file or files, translates them into executable machine code, and produces an executable file.
    • Compilers perform static analysis of the code, detecting errors and optimizing the code for performance.
    • Examples include GCC (GNU Compiler Collection) for C/C++, Java Compiler for Java, and Swift Compiler for Swift.

In SHORT, IDEs provide a comprehensive development environment with tools for coding, debugging, and building applications. Interpreters execute code directly without compilation. Code editors are lightweight tools for editing source code. Compilers translate source code into machine code or bytecode for execution. Each serves a different purpose in the software development process.

What is Node.js and How to Install It?

Node.js is a JavaScript runtime that allows you to run JavaScript code outside the browser. Technically, you can run JavaScript on the browser itself, but having Node.js installed on your machine is very useful because it allows you to install third-party libraries and packages that help with development.

Here’s how to install and set up Node.js:

  1. Install Node.js:
    • Go to the official Node.js website: https://nodejs.org/en.
    • Download and install Node.js for your operating system. The installation process is straightforward.
  2. Set Up a Project in VS Code:
    • After installing Node.js, create a new folder for your project.
    • Open the folder in VS Code either by right clicking the folder and selecting "Open with Code," or by dragging and dropping the folder into VS Code.
  3. Create an HTML File:
    • In VS Code, go to the Explorer tab (the first tab on the left sidebar).
    • Create a new file called index.html
    • If you're not familiar with HTML, don't worry. Just follow these steps:
      • In the newly created index.html file, press Shift + 1 (which gives you an exclamation mark!) and then press Tab. This will automatically generate a basic HTML template

JavaScript First Program

To write your first JavaScript program, let's break it down step by step:

  1. Where do we write JavaScript in an HTML file?
    • We use the <script> element to include JavaScript code in an HTML file.
    • There are two common places where we can add the <script> element:
      • Inside the <head> section
      • Inside the <body> section, usually at the end before the </body> tag.
    • Best Practice: It’s generally recommended to place the <script> at the end of the <body>. This is because the browser reads and processes HTML from top to bottom.If you put the script in the <head>, the browser will pause rendering the page to execute the JavaScript, which can make the page appear blank for a moment.By placing the script at the end of the body, you ensure the HTML content is fully loaded before the JavaScript runs.
  2. Why put JavaScript at the end of the <body>?
    • JavaScript often interacts with elements on the page, like hiding or showing content. By putting the script at the end, you're making sure all the HTML elements are fully rendered before JavaScript tries to interact with them.
    • If you place JavaScript in the <head>, it can delay rendering the page, which may lead to a bad user experience as the page appears blank until the script is finished.
  3. Exception: Sometimes, third-party scripts (like analytics tools) might need to be placed in the <head>, but this is an exception, not the rule
  4. First JavaScript Program: Now, let’s write your first JavaScript program. This is a simple line of code that displays a message in the browser's console:
    <script>console.log("Hello NaseebCodeStudio")</script>
    • Here, console.log("Hello NaseebCodeStudio"); is a statement. A statement is a single line of code that performs an action. In this case, it logs (prints) the message "Hello NaseebCodeStudio" to the browser’s console.
    • Statements in JavaScript usually end with a semicolon “;” to mark the end of the instruction.
  5. How to see the result in the browser:
    • After saving your changes, open your HTML file in a browser.
    • Right-click anywhere on the page and select "Inspect" (shortcut: Ctrl + Shift + I).
    • Go to the "Console" tab, and you’ll see the message "Hello NaseebCodeStudio" displayed there.

This is how you can start with a simple JavaScript program and view its output!

What is Window?

In JavaScript, the `window` object represents the browser window or tab that contains the JavaScript code. It is a global object, which means it is accessible from anywhere in your JavaScript code.

We can write these methods directly without using window keyword.

Here are some important and commonly used methods and properties of the `window` object:

window.alert()

Displays an alert dialog box with a message and an OK button. 

window.alert("Hello, world!");

window.prompt()

Displays a prompt dialog box with a message, an input field for the user to enter text, and OK and Cancel buttons. 

let userInput = window.prompt("Please enter your name:");

window.confirm()

Displays a confirmation dialog box with a message, OK, and Cancel buttons. Returns true if the user clicks OK and false if they click Cancel. 

let userConfirmation = window.confirm("Are you sure you want to proceed?");

window.open()

Opens a new browser window or tab with the specified URL 

window.open("https://www.example.com", "_blank");

window.close()

Closes the current browser window. 

window.close();

window.location

Provides information about the current URL and allows you to navigate to a new URL. 

console.log(window.location.href); // Current URL
window.location.href = "https://www.example.com"; // Redirect to a new URL

window.print()

prints the complete document;

window.print();

These are just a few examples of the methods and properties provided by the `window` object in JavaScript. The `window` object also provides access to other browser-related functionalities, such as working with cookies, handling events, and interacting with the browser's history and navigation.

JavaScript Comments

A comment is a note or explanation written in the code that is not meant to be executed by the computer. It helps make the code more understandable for anyone reading it, including the person who wrote it! You can think of it by leaving reminders, explanations, or descriptions about how the code works or why certain things are done in a specific way.

Types of Comments

There are two main types of comments:

Single Line Comment:

  • This type of comment is used for brief explanations or notes that fit on one line.
  • In most programming languages (like JavaScript, HTML, and CSS), you add // before the comment. 
// This is a single-line comment
let name = "Naseeb Khan";

Shortcut key in vs code:
Control + / or Control + K + C

Multi-line Comment

  • This type of comment is used when you need to explain something in more detail, spanning multiple lines.
  • You start with /* and end with */. Everything between these symbols is ignored by the computer.
/* This is a multi-line comment.
It can cover several lines. */
let age = 24;

Why Use Comments?

  • Make Code Understandable: Comments help you and others understand what each part of your code is doing.
  • Provide Clarity: When a piece of code is complicated or unusual, you can use comments to explain why you wrote it that way
  • Easier to Maintain: If you return to the code after a long time, comments make it easier to remember how things work.
  • Team Collaboration: When working with a team, comments help other team members understand your thought process

Separation of concern

When we write a lot of JavaScript code sometimes thousands or millions of lines it’s important to keep it separate from our HTML. This means putting the JavaScript code into its own file, usually named script.js, and then linking it to our HTML file using this line: 

<script src="script.js"></script>

Why do we separate them?

Think of your home: you keep your food and cooking tools in the kitchen, but you don’t keep your bed there. Each part of your house has its own purpose. Similarly, HTML is all about the content of the webpage, while JavaScript is about how the webpage behaves.

JavaScript is like the instructions that tell the browser how to make the page interactive and dynamic. By keeping the HTML and JavaScript separate, we can manage and organize our code better, making it easier to read, update, and maintain

  3. Writing JavaScript Code in CMD

Word VS Keyword

A word is just like a normal English word or a word from any other language that we use for communication.
A keyword, however, is a special word in programming that has a predefined meaning and is used to perform specific actions in the language.

For example, in JavaScript, if, else, and function are keywords.
You cannot use them as variable names because they already have special meanings.
