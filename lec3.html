CONTAINERS / VARIABELS

In programming, variables are one of the most important concepts in JavaScript (JS) and in any other programming language. A variable is like a container that stores data temporarily in a computer's memory, making it easier to access and manage that data later.
You can think of variables like boxes in your kitchen where you store different ingredients such as salt or sugar, and you label each box according to what's inside. Similarly, in programming, we store data in a location and give that location a name, so we can use that name to retrieve the data in the future.

How to Declare a Variable?

Before the ES6 update in JavaScript, variables were declared using the var keyword. However, this caused some issues which we will discuss later in the course. After ES6, it became a good practice to use the let keyword to declare variables. To declare a variable, you need to:

  1. Use the let, var, const keyword.
  2. Give the variable a name (also called an "identifier").
  3. Optionally, assign a value to it

The line ends with a semicolon ;.

let name;
console.log(name); // Output: undefined

By default, if you don't assign a value to a variable in JavaScript, it will be undefined

You can also assign a value at the time of declaring the variable: 

let name = "NaseebKhan";
console.log(name); // Output: NaseebKhan

In this example, we declared a variable using let, named it name, and assigned the string "NaseebKhan" to it.

Rules for Naming Variables

There are some important rules to follow when naming variables in JavaScript:

1. No reserved keywords: You cannot use JavaScript reserved keywords like if, var, const, let, etc., as variable names. 

let if = 'NaseebKhan'; // This will give an error.

2. Meaningful names: Always use meaningful names for variables so it's clear what they represent. 

let a; // Not meaningful
let $; // Not meaningful
let _; // Not meaningful
let firstName; // Meaningful

3. Cannot start with a number: A variable name cannot begin with a number 

let 1name; // Invalid

back to the rule no 2 1name is meaningless.

4. No spaces or hyphens: Variable names cannot contain spaces or hyphens. If you have multiple words in a variable name, use camelCase or snake_case. 

et firstName; // camelCase (preferred in JavaScript)
let first_name; // snake_case (less common in JavaScript)

5. Case-sensitive: Variable names are case-sensitive, which means name and Name are considered two different variables: 

let name;
let Name;

Declaring Multiple Variables

There are two ways to declare multiple variables:

1. Single line: You can declare multiple variables in a single line separated by commas. 

let firstName = "Naseeb", lastName = "Khan";

2. Multiple lines (modern best practice): It's better practice to declare each variable on its own line for clarity.

let firstName = "Naseeb";
let lastName = "Khan";

By following these rules and best practices, you will write cleaner and more readable code.

Constant

In JavaScript, we use the let or const keywords to declare variables.

Using let: When you declare a variable with let, it means you can change its value later

let interestRate = 0.3;
interestRate = 1; // Now interestRate is 1

You can assign a new value to interestRate because you used let.

Using const: In some cases, you might not want the value of a variable to change, because if it changes, it can create bugs or errors in your program. In such cases, you use const to declare the variable, which makes it a constant. This means the value cannot be changed once it's set. 

const interestRate = 0.3;
interestRate = 1; // This will make an error

In this example, trying to change the value of interestRate will result in an error, because it's declared with const.

Best Practice:

  • Use const when you know the value of the variable should never change.
  • Use let if you need to change the value of the variable later.

This helps you avoid errors and bugs in your code by preventing unwanted changes to important values.

Declaration VS Initialization

Declaration means creating a variable.

Initialization means giving a value to a variable for the first time.

The var keyword (from ES5) has some issues: it becomes part of the window object, and it allows redeclaration and reassignment multiple times, which can cause bugs.

Primitive types

Primitive types are the basic building blocks in JavaScript. We've already learned how to declare and initialize variables, but you might be wondering what types of values we can store in them. You've seen how to use strings, but JavaScript offers more data types.

Their values can contain only a single thing (be it a string or a number or whatever)

There are two main categories of data types in JavaScript:

  1. Primitive (or Value) Types
  2. Reference (or Object) Types

In this section, we'll focus on primitive types. These include:

  • String: A sequence of characters.
  • Number/BigInt: Represents both integers and floating-point numbers.
  • Boolean: Can be either true or false.
  • Undefined: A variable that hasn't been given a value yet.
  • Null: Represents an empty or non-existent value.
  • Symbol: for unique identifiers. (helpful in react)

Let’s break it down with some examples:

String

sequence of text characters covered by single quotes ‘ or double quotes “

let name = "NaseebKhan"; // This is a string literal with double quotes.
let name = 'NaseebKhan'; // This is also a string literal with single quotes.
let name = `NaseebKhan`; // This is another string literal with back tic

Number

Any numerical value, either integer or floating-point. 

let age = 24; // This is a number literal.

BigInt

JavaScript's number type can’t safely represent integers larger than (253 - 1) which equals 9007199254740991 or smaller than -(253 - 1).

The number type can store larger integers (up to 1.7976931348623157 * 10308), but outside the safe integer range, precision is lost.To represent such large numbers precisely, we need to use the BigInt type. In BigInt, we append "n" to the end of the number, like 1234566356534543554345n.

These will result in precision loss due to exceeding the safe integer range:

console.log(9007199254740991 + 1); // 9007199254740992
console.log(9007199254740991 + 2); // 9007199254740992 (incorrect due to precision
loss)

To correctly handle larger integers, use BigInt:

console.log(9007199254740991n + 2n); // 9007199254740993n
console.log(Infinity); // infinity is a number which is larger than any number

Boolean

This type has only two values: true or false and is typically used for logical operations.

let isApproved = true; // This is a Boolean literal.

You would use Booleans, for example, to track if an order is approved, where the value can either be true (approved) or false (not approved). true and false are reserved keywords, so you cannot use them as variable names.

Undefined

A variable that has been declared but not initialized automatically gets the undefined value. 

let firstName; // Since we didn't assign a value, it's undefined by default.
let firstName = undefined; // both are the same

You can also explicitly set a variable to undefined, but this is not common practice

Null

This is used when you want to intentionally clear or reset a variable to "nothing." 

let lastName = null; // This explicitly sets the variable to null.

Think like you have a store website or app where a user hasn’t selected a product yet, you might initialize the variable as null. Later, when they select, you can assign the chosen product to this variable. 

let selectedProduct = null;

In ES6, JavaScript introduced another primitive type called Symbol.

These are the basic primitive types in JavaScript, and they form the foundation for handling simple data in your programs.

Symbol

In JavaScript, the Symbol data type is used to create unique identifiers. Even if two symbols are created with the same description, they are guaranteed to be unique. 

const name = Symbol('Naseeb'); // Create a symbol with the description 'Naseeb'
const anotherName = Symbol('Naseeb'); // Another symbol with the same description

// Compare the two symbols
console.log(name === anotherName); // Output: false

Here, even though both name and anotherName have the same description ('Naseeb'), they are unique symbols. That’s why the comparison name === anotherName returns false. This ensures that symbols can be used as unique identifiers.

Dynamic Typing

Dynamic typing in JavaScript means that the type of variable can change while the program is running. This is different from statically typed languages, where the type of variable is fixed when you declare it and cannot change later

Static vs Dynamic Typing

1. Static Typing (Statically Typed Languages):
In languages like C++ or Java, when you create a variable, you must define its type at the time of declaration, and that type cannot change. 

string name = "Naseeb"; // This variable is always a string.

2. Dynamic Typing (Dynamically Typed Languages):
In JavaScript (a dynamic language), the type of variable can change at runtime, meaning you can first assign a string, and later change it to a number. 

let name = "Naseeb"; // At first, it's a string.
name = 24; // Now, it’s a number.

In short, dynamic typing allows flexibility in JavaScript by letting the type of a variable change during program execution.

Checking Variable Type

In JavaScript, you can check the type of a variable using the typeof unary operator.

let name = "Naseeb";
console.log(typeof(name)); // Output: "string"

let arr1 = [1, 2, 3];
console.log(arr1 instanceof Array); // true

Does arr1 belongs to Array, instanceof works with only reference data types.

Now, if you change the value of name to a number:

name = 24;
console.log(typeof(name)); // Output: "number"

This is what makes JavaScript a dynamically typed language. The type of variable is determined at runtime based on the value it holds.

Numbers in JavaScript

In JavaScript, there is only one type for numbers, whether they are whole numbers (integers) or decimal numbers (floating points). Both 24 and 24.5 are considered numbers.

let number1 = 24; // This is a number.
let number2 = 24.5; // This is also a number.

Special Types: undefined and null

undefined: This is both a type and a value. It represents a variable that has been declared but not assigned a value yet. 

let x;
console.log(typeof(x)); // Output: "undefined"

null: This represents an intentional absence of any object value. Interestingly, the typeof null in JavaScript returns "object." 

let y = null;
console.log(typeof(y)); // Output: "object"

NaN: Represents a data where a number is calculated with any non-number data: 

let y = 10; // number
let x = “Naseeb”; // non-number
let z = y / x;
console.log(typeof(z)); // Output: "NaN"

Type Conversion

When working with JavaScript, many operators and functions automatically change the type of value you give them to match what they need.

  • The alert() function converts any value to a string because it displays text.
  • Math operations (like addition or subtraction) convert values to numbers so calculations can happen.

But sometimes, you need to manually convert a value to a specific type

Converting to a String (String Conversion)

String conversion is used when we want to turn a value into text (a string).

  • The alert() function automatically converts a value into a string to display it.
  • You can also manually convert a value to a string using the String() function.

Here’s how it works:

let value = true;
alert(typeof value); // Outputs: "Boolean" (the type of value is Boolean)

value = String(value); // Converts the value to a string
alert(typeof value); // Outputs: "string" (now the type is string)

How does it convert?

  • true becomes "true" (a string).
  • false becomes "false".
  • null becomes "null".

String conversion is generally straightforward and works as expected.

Numeric / Number Conversion

When working with numbers in JavaScript, values can be automatically or explicitly converted into numbers depending on the context. Here's a simpler breakdown of how this works:

Automatic Conversion in Math Operations

JavaScript often converts strings to numbers automatically when performing mathematical operations.

alert("6" / "2"); // Result: 3 (strings "6" and "2" are automatically converted to numbers)

Explicit Conversion to Numbers

To manually convert a value into a number, you can use the Number(value) function.

let str = "123"; // This is a string
alert(typeof str); // Output: "string"

let num = Number(str); // Convert the string "123" to a number
alert(typeof num); // Output: "number"

Explicit conversion is particularly useful when you're getting input from a text field or other sources where data is in string format, but you need to work with numbers.

What Happens with Invalid Conversions?

If you try to convert something that can't be understood as a number (like random text), the result is NaN (Not a Number). 

let age = Number("this is not a number");
alert(age); // Output: NaN (conversion failed)

Rules for Numeric Conversion:

Here’s what happens when different types of values are converted to numbers:

Value Converted To
undefined NaN
null 0
true / false 1 (true) and 0 (false)
String (e.g., "123") Whitespace at the start or end is ignored. If valid, it's converted 
alert(Number(" 123 ")); // Result: 123 (whitespace is ignored)
alert(Number("123z")); // Result: NaN (error due to "z")
alert(Number(true)); // Result: 1
alert(Number(false)); // Result: 0

Note:

  • null converts to 0.
  • undefined converts to NaN.

Math Operators and Numeric Conversion

Most math operations (like addition, subtraction, division) will automatically convert non-number values to numbers before performing the calculation.

Boolean Conversion

Boolean conversion is a simple concept in JavaScript. It determines whether a value is true or false.

This happens automatically in logical operations (like if conditions) but can also be done explicitly using Boolean(value).

Here’s the rule:

  • “Empty” values (falsy values) like 0, an empty string "", null, undefined, and NaN are treated as false.
  • Everything else is treated as true.
alert(Boolean(1)); // true (1 is not empty)
alert(Boolean(0)); // false (0 is an empty value)

alert(Boolean("hello")); // true (non-empty string)
alert(Boolean("")); // false (empty string)

alert(Boolean("0")); // true (non-empty string, even though it’s just "0")
alert(Boolean(" ")); // true (a string with spaces is still non-empty)

Important:

Some programming languages (like PHP) consider "0" as false. However, in JavaScript, any non-empty string including "0" or " " (spaces) is true.

JavaScript often needs to convert one type of value into another. The three most common conversions are:

  • Happens when we display a value (e.g., alert, console.log).

2. Numeric Conversion

  • Rules for conversion:

  • Happens in logical operations or explicitly with Boolean(value).
  • Rules for conversion:

Key Points to Remember

  1. Falsy values: 0, null, undefined, NaN, and "" are false.
  2. Truthful values: Anything else is true, including "0" and " " (a string with spaces).
  3. Common mistakes:

Reference Data Type

Now that we've explored all the basic (primitive) types, let's dive into reference types. In the reference category, we have objects, arrays, and functions. But what exactly is an object?

Object

An object in JavaScript, or in other programming languages, is like real-world objects. Think of a person. A person has a name, age, address, etc. These are called properties of a person. We use the same concept in JavaScript. When we have multiple related pieces of information, instead of creating separate variables, we can put them together inside an object.

Example, instead of writing: 

let name = "Naseeb";
let age = 23;

Both name and age are related to a person. Rather than declaring two separate variables, we can create a single object to represent the person, making our code cleaner. Here’s how you can create an object: 

let person = { name: "Naseeb", age: 23 };

These curly braces {} are called object literals. Inside the object literal, we add one or more key-value pairs, where keys represent the properties of the object. In this example, name and age are the keys, and "Naseeb" and 23 are the values. Each key-value pair is separated by a comma.

To print this object, we can use:

console.log(person);

There are two ways to work with the properties of an object

1. Dot notation: This is the more common way to access or change properties. For example, to change the name of the person:

person.name = "Khan";
console.log(person.name); // print only a name of person which is now khan

2. Bracket notation: Another way to access properties is by using brackets, passing the property name as a string. For example: 

person['name'] = 'NK';
console.log(person.name);

Both methods work, and you can choose based on what feels best for your code.

Array

In everyday applications, we often work with lists of objects. For example, you might have a list of products in a shopping cart or a list of colors that a user has selected. In these situations, instead of creating separate variables for each item, which would be inconvenient, we use an array to store everything in one place.

Imagine you have a list of colors. Instead of creating a new variable for each color, we can group them into a single variable using an array. 

let selectedColors = [ ];

The square brackets ( [ ] ) represent an empty array, which means there are no items in it yet. We can add items to this array like this: 

let selectedColors = ['red', 'green', 'blue', ‘black’];

Now, if we print the array using console.log(selectedColors);, we get the list of colors. Each item in the array has a position number, called an index, that starts from 0. To access specific items, we use their index.

to print the first color:

console.log(selectedColors[0]); // Output: 'red'

JavaScript is a dynamic language, and arrays are also dynamic. This means we can change the size of the array and add new items to it. For example, to add a new color:

selectedColors[4] = 'black';

Now the array has five items, and the fifth one is 'black'. Unlike other programming languages, where all items in an array must be the same type, in JavaScript, we can store different types of data in a single array: 

let mixedArray = [1, 'a', true, null];

Arrays in JavaScript are technically objects and have built-in properties that we can access. For example, to find the length of an array (the number of items it contains), we use the .length property:

console.log(selectedColors.length); // Output: 5

In summary, an array is a data structure that allows us to store and manage a list of items efficiently.

Function

In JavaScript, functions are one of the core building blocks. They are like small programs that you can use over and over to perform a specific task or calculate something. You can think of a function as a set of instructions that tell the computer what to do.

What is a Function?

A function in JavaScript (or in programming in general) is a collection of statements that perform a specific task. Functions allow us to write code once and run it multiple times, potentially with different inputs. This means we can make our code reusable and more efficient.

Creating a Function

To create a function, you use the function keyword, followed by a name, parentheses (), and curly braces {}. Inside the curly braces is the body of the function, where you write the logic or instructions.

Example, a simple function that prints a message to the console:

function greetings() {
console.log("Hello NaseebCodeStudio");
}

Here, we’ve declared a function, but it won't do anything until we call it by its name:

greetings(); // This will print "Hello NaseebCodeStudio" in the console

Functions with Inputs (Parameters)

A function can take inputs, called parameters. These parameters allow you to pass different values to the function so it can act on them. For example, let’s create a function that greets a person by name: 

function greeting(name) {
 console.log("Hello " + name);
}

When we call this function, we can pass different names to get different results:

greeting("Naseeb"); // Prints "Hello Naseeb"
greeting("Khan"); // Prints "Hello Khan"

In this example, name is the parameter, and "Naseeb" or "Khan" are the arguments passed when calling the function.

Parameters and Arguments

  • Parameters are like placeholders or variables inside the function that store the values you pass on when calling the function.
  • Arguments are the actual values you pass to the function when you call it.

For example, in greeting(name), name is the parameter, and when you call greeting("Naseeb"), "Naseeb" is the argument that gets passed to the name parameter.

Multiple Parameters

A function can also take more than one parameter. For instance:

function greet(name, lastName=”Khan”) {
 console.log("Hello " + name + " " + lastName);
}

A function with parameters which holds default value “khan”, 

function greet(name, lastName) {
 console.log("Hello " + name + " " + lastName);
}

Here, the function takes two parameters, name and lastName, and you can pass two arguments when calling it: 

greet("Naseeb", "Khan"); // Prints "Hello Naseeb Khan"

In this case, name holds the value "Naseeb" and lastName holds the value "Khan"

Overview:

  • A function is a block of code that performs a specific task.
  • You define a function using the function keyword, a name, and a body inside curly braces.
  • Functions can take parameters (inputs) that control how the function behaves.
  • Arguments are the actual values you pass when calling the function.
  • Functions help make your code reusable by allowing you to run the same code with different inputs.

What is the return keyword in JavaScript?

The return keyword is used in a function to send back a result or value to wherever the function was called. It essentially gives back the final output of the function to be used elsewhere in the code. 

function square(number) {
return number * number;
}

In the example above, the function square takes a number, multiplies it by itself, and then sends that result back using the return keyword.

How to Use return:

To make use of the result, we need to store the returned value in a variable:

let result = square(2); // result will be 4
console.log(result); // This prints 4

Alternatively, we can directly use the function inside console.log() without storing it in a variable:

console.log(square(2)); // This also prints 4

Here, the JavaScript engine first calls the square function, gets the returned result (4), and then passes that value to console.log() to print it.

How Many Functions Were Called?

in this example: 

console.log(square(2));

Two functions were called:

  1. The square function, which calculates and returns the square of the number.
  2. The console.log() function, which prints the result to the console.

Real-Life Use of Functions

In real-life applications, there can be thousands of functions working together. Each function is responsible for a specific task, and they often return values that are passed to other functions to create complex logic or functionality.
