CHAPTER 11

EVENT

Event = something that happens in the browser

Examples:

  • click
  • typing
  • scrolling
  • mouse move

SIMPLE DEFINITION Event = user action or browser action

Real Life Analogy

Doorbell

  • someone presses button → event
  • you open door → response

Same in JS:

  • user clicks → event
  • your code runs → handler

EVENT HANDLING

Event handling = what to do when event happens

Example:

const btn = document.querySelector("button");

btn.addEventListener("click", () => {
  console.log("Button clicked!");
});

BREAK IT DOWN

addEventListener Means "listen for this event"

"click" is Event Type

() => {} Function = what to do

EVENT LISTENER (IMPORTANT)

Event Listener = a watcher it waits for event and then runs code.

ADD EVENT (MAIN METHOD)

// Standard Way
element.addEventListener("event", handler)

// Example: 
const box = document.querySelector(".box");

box.addEventListener("mouseover", () => {
  box.style.backgroundColor = "blue";
});

❌ OLD WAY (Avoid)

box.onclick = function() {};

Problem:

  • only one event allowed
  • overrides previous

REMOVE EVENT (IMPORTANT)

Sometimes you need to stop listening

Rule: same function reference needed

function handleClick() {
  console.log("Clicked!");
}

btn.addEventListener("click", handleClick);

// remove it
btn.removeEventListener("click", handleClick);

❌ WRONG (Common mistake)

btn.removeEventListener("click", () => {})
  • Doesn't work
  • Because function is different

Engineering Rule

Always store handler in variable/function if you plan to remove it 

// REAL EXAMPLE (ADD + REMOVE)
const btn = document.querySelector("button");

function sayHello() {
  console.log("Hello!");
}

btn.addEventListener("click", sayHello);

// stop after 5 seconds
setTimeout(() => {
  btn.removeEventListener("click", sayHello);
}, 5000);

EVENT OBJECT (VERY IMPORTANT)

btn.addEventListener("click", (e) => {
  console.log(e);
})

"e" Gives:

  • target element
  • mouse position
  • key pressed
// Most used:
e.target // Which element triggered event

Final Mental Model

  • Event = Something happens
  • Listener = waiting for it
  • Handler = what you do

Types of Events

Instead of learning 100 events, we focus on the MOST USEFUL ones used in real projects.

HOW TO THINK ABOUT EVENTS

Events are grouped like this:

Category What it means
Mouse Events click, hover
Keyboard Events Typing
Form Events input, submit
Window events load scroll

MOUSE EVENTS (MOST USED)

// 1. click: user clicks something 
btn.addEventListener("click", () => {
  console.log("Clicked!");
});

// 2. dblclick: double click 
btn.addEventListener("dblclick", () => {
  console.log("Double clicked!");
});

// 3. mouseover: mouse enters element 
box.addEventListener("mouseover", () => {
  box.classList.add("hover");
});

// 4. mouseout: mouse leaves element 
box.addEventListener("mouseout", () => {
  box.classList.remove("hover");
});

Pro tip: for hover effects prefer CSS, Use JS only if logic needed.

Keyboard events

// 1. keydown: when key is pressed 
document.addEventListener("keydown", (e) => {
  console.log(e.key);
});

// 2. keyup: when key is released 
document.addEventListener("keyup", () => {
  console.log("Key released");
});

// keyup example:
input.addEventListener("keydown", (e) => {
  if (e.key === "Enter") {
    console.log("Submitted!");
  }
});

Form Events (very important)

// 1. input: when user types
input.addEventListener("input", (e) => {
  console.log(e.target.value);
});

input.addEventListener("input"), (e) => {
  console.log(e.data);
}

// 2. change: when input loses focus AND value changed 
input.addEventListener("change", () => {
  console.log("Changed");
});

select.addEventListener("change", (e) => {
  console.log(e.target.value);
});

// Example program 
let button = document.querySelector("button");
const fileInput = document.querySelector("input");

button.addEventListener("click", function(){
  fileInput.click(); // Transform the click to input 
});

fileInput.addEventListener("change", function(e){
  const file = e.target.files[0];
  if (file){
    button.textContent = file.name;
  }
})

// 3. submit: form submit 
form.addEventListener("submit", (e) => {
  e.preventDefault(); // stop reload
  console.log("Form submitted");
});

Window Events

// 0. keydown 
window.addEventListener("keydown", function (e){
  if (e.key === " "){
    h1.textContent = "space";
  }else{
    h1.textContent = e.keys;
  }
})

// 1. Load: when page fully loads 
window.addEventListener("load", () => {
  console.log("Page loaded");
});

// 2. scroll: when user scrolls 
window.addEventListener("scroll", () => {
  console.log("Scrolling...");
});

// JavaScript 
const select = document.querySelector("select");
const h1 = document.querySelector("h1");
select.addEventListener("change", function(e){
  h1.textContent = "Device Selected"; // replace h1 text 
  h1.textContent = `${e.target.value} Selected`;  // replace h1 text with selected value
  console.log(e.target.value); // returns selected value 
})

const form = document.querySelector("form");
const inputs = document.querySelectorAll("input");

form.addEventListener("submit", (e) => {
  e.preventDefault(); 

  // creating elements & adding classes
  const card = document.createElement("div");
  card.classList.add("card");

  const profile = document.createElement("div");
  profile.classList.add("profile");

  const h2 = document.createElement("h2");
  h2.textContent = inputs[0].value;
  
  const img = document.createElement("img");
  img.setAttribute("src", inputs[1].value);

  const h4 = document.createElement("h4");
  h4.textContent = inputs[2].value;

  const p = documnet.createElement("p");
  p.textContent = inputs[3].value;

  // append childs into corresponding parents 
  profile.appendChild(img);
  card.appendChild(profile); 

  card.appendChild(h2);
  card.appendChild(h4);
  card.appendChild(p);

  // Make inputs empty
  inputs.forEach((inputElement) => {
    if (inputElement.type !== "submit"){
      inputElement.value = "";
    }
  })
});
