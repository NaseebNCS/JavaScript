Control Attributes

Attributes = extra information inside HTML elements.

<input type="text" id="username" value="Naseeb Khan" class="box">

Here:

  • type="text is attribute
  • id="username" is attribute
  • value="Naseeb Khan" is attribute
  • class="box" is attribute

Simple:
Attributes = settings of an element

Why Manipulate Attributes

To:

  • change values
  • add new attributes
  • remove attributes
  • check attributes
// 1. Get Attribute
let input = document.getElementById("username");

console.log(input.getAttribute("value"));
// Output: Naseeb Khan

// 2. Set / Change Attribute 
input.setAttribute("value", "NaseebCodeStudio");

// Now: Value="NaseebCodeStudio"

// 3. Add New Attribute
input.setAttribute("placeholder", "Enter your name");

// Adds: placeholder="Enter your name"

// 4. Remove Attribute
input.removeAttribute("class");

// Removes: class="box"

// 5. Check if Attribute Exists
console.log(input.hasAttribute("id"));

// Output: true

// 6. Get All Attributes 
console.log(input.attributes);

// Returns Collection like: 
// type="text"
// id="username"
// value="Ali"
// class="box"

// LOOP through attributes 
for (let attr of input.attributes){
  console.log(attr.name + " = " + attr.value);
}

Direct Property vs Attribute (IMPORTANT)

<input id="box" value="Naseeb">

let el = document.getElementById("boxy");

// Attribute 
console.log(el.getAttribute("value"));

// Property 
console.log(el.value);
Attribute Property
HTML value Live value
Static Can Change dynamically 
// Example  
el.value = "Changed!";

Now:

  • el.value is "Changed!"
  • getAttribute("value") is still "Naseeb"

Simple:
Attribute = original
Property = current/live 

// Example:
<img id="img" src="image1.jpg">

let img = document.getElementById("img");

// Change image
img.setAttribute("src", "image2.jpg");

// Check
console.log(img.getAttribute("src"));

Image changes on screen

Special Attributes (Shortcut way)

Instead of:

input.setAttribute("value", "NaseebCodeStudio");

// You can do:
input.value = "NaseebCodeStudio";

Common shortcuts:

input.id
input.className
input.value
img.src
a.href

Important Rule

Use:

  • setAttribute() used for custom attributes
  • direct property used for common ones

In Simple Words:

  • Attributes = element settings
  • getAttribute → read
  • setAttribute → change/add
  • removeAttribute → delete
  • hasAttribute → check
  • attributes → list all
  • property ≠ attribute

Easy Analogy:
Element = mobile phone 📱
Attributes = settings (brightness, volume, name)
You can:

  • change settings
  • remove settings
  • check settings

Engineer Tip

In real Projects:

  • Use .value, .src, .href (faster)
  • Use setAttribute() for custom stuff

Control Elements

Dynamic DOM manipulation means:
Creating, adding, and removing elements using JavaScript at runtime

💡 Real apps (YouTube, Facebook, Todo apps)
👉 don’t have static HTML they build UI dynamically

Think like this:
👉 DOM = LEGO blocks 🧱
👉 JavaScript = your hands ✋

You can:

  • create new block
  • attach it anywhere
  • remove it anytime

CREATE ELEMENT

const div = document.createElement("div");

👉 This creates element in memory only
👉 NOT visible yet ❌

Add content

div.textContent = "Hello Naseeb";

Add class (best practice)

div.classList.add("box");

ADDING ELEMENTS TO DOM

appendChild (classic)

parent.appendChild(div);

// 👉 Adds element at the end

append (modern & flexible)

parent.append(div);

Difference:

  • appendChild → only elements
  • append → text + multiple items

PREPEND (VERY IMPORTANT)

parent.prepend(div);

// 👉 Adds element at the start

VISUAL UNDERSTANDING

// Before:
<ul>
  <li>Item 1</li>
</ul>


// After append:
<ul>
  <li>New Item</li>
  <li>Item 1</li>
</ul>


// After prepend:
<ul>
  <li>New Item</li>
  <li>Item 1</li>
</ul>

COMPLETE REAL EXAMPLE

const list = document.querySelector("ul");

// create element
const li = document.createElement("li");

// add content
li.textContent = "Learning DOM from NaseebCodeStudio";

// add class
li.classList.add("task");

// append to list
list.append(li);

REMOVING ELEMENTS

// Simple remove
li.remove();

// Remove from parent
parent.removeChild(li);
// 👉 Old but still used sometimes

REAL PROJECT (Add + Remove Todo)

const input = document.querySelector("#input");
const btn = document.querySelector("#btn");
const list = document.querySelector("#list");

btn.addEventListener("click", () => {
  const value = input.value.trim();
  if (!value) return;

  const li = document.createElement("li");
  li.textContent = value;

  list.append(li);
  input.value = "";
});

// remove item (PRO WAY)
list.addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    e.target.remove();
  }
});

ENGINEERING MINDSET

❌ Beginner Thinking: 👉 “Just create and append”
✅ Engineer Thinking: 👉 “How will this scale with 1000 elements?”

BEST PRACTICES

1. Avoid Multiple DOM Access: ❌ Bad:

document.querySelector("ul").append(li);

✔️ Good:

const list = document.querySelector("ul");
list.append(li);

2. Use Document Fragment (PERFORMANCE 🔥): For multiple elements:

const fragment = document.createDocumentFragment();

for (let i = 0; i < 5; i++) {
  const li = document.createElement("li");
  li.textContent = "Item " + i;
  fragment.append(li);
}

list.append(fragment);

WHY?

  • reduces reflow
  • faster rendering

Control CSS

You can change styles in JavaScript in 2 main ways:

  1. .style → direct styling
  2. classList → CSS-based styling (PRO way)

.style (Direct Styling)

Think like this: i'm directly painting the element with JS

Basic Example:

const box = document.querySelector(".box");

box.style.color = "red";
box.style.backgroundColor = "black";

Important Rule:

CSS → background-color (kabab-case)

JS → backgroundColor (camelCase)

Real Example:

box.style.width = "200px";
box.style.padding = "10px";
box.style.borderRadius = "10px";

Problems with .style

  • ❌ Hard to manage in big apps
  • ❌ Mixes design with logic
  • ❌ Not reusable

That’s why professionals avoid it for major styling

classList (PROFESSIONAL WAY)

Think like this: "JS controls behavior, CSS controls design" 

// CSS
.red {
  color: red;
}

.active {
  background: black;
  color: white;
}

// JavaScript
const box = document.querySelector(".box");

box.classList.add("red");

classList Methods (MASTER THESE)

// 1. add()
box.classList.add("active"); // add class

// 2. remove()
box.classList.remove("active"); // remove class 

// 3. toggle()
box.classList.toggle("active"); // if class exist → remove if not → add

// 4. contains()
box.classList.contains("active"); // Check if class exists

.style vs classList (IMPORTANT)

Feature .style classList
Use case small changes real apps
Maintainable ❌ ✅
Reusable ❌ ✅
Clean Code ❌ ✅

ENGINEERING RULE (REMEMBER THIS)

Use .style for: quick dynamic values (width, height, position)
Use classList for: UI states (active, hidden, dark, open)

Best Practices

// 1. NEVER mix logic + design
// ❌ Bad:
box.style.color = "red";

// ✔️ Good:
box.classList.add("error");

// 2. Use meaningful class names
// ✔️ active, hidden, open, error
// ❌ redBox123

// 3. Think in STATES
// UI = states
    // active / inactive
    // open / closed
    // visible / hidden

// 4. Keep CSS powerful
// Let CSS handle:
    // animations
    // colors 
    // layout 

// 5. Avoid inline styles in large apps

Final Mental Model

.style = quick control
classList = real engineering solution
