Timer

Timers and intervals are used everywhere in real apps:

  • countdown timers
  • auto sliders
  • notifications
  • games
  • loading systems
  • deboince/throttle

If DOM gives life to UI, then timers control TIME itself in JavaScript.

1. FIRST UNDERSTAND JAVASCRIPT NATURE

JavaScript is single-threaded, meaning it can only execute one task at a time. Timers and intervals allow us to schedule tasks to run later or repeatedly without blocking the main execution thread.

JavaScript normally runs: Line by Line

Example:

console.log("One");
console.log("Two");
console.log("Three");

// Output:
// One
// Two
// Three

BUT WHAT IF YOU WANT:

Run this After 3 seconds OR Run this every second

that's where timers come in.

2. TWO MAIN TIMER FUNCTIONS

Function Purpose
setTimeout run ONCE after a delay
setInterval run REPEATEDLY at regular intervals / run again and again

3. setTimeout() (RUN ONCE)

setTimeout schedules a function to run once after a specified delay in milliseconds.

Basic Syntax: 

setTimeout(function, delay, arg1, arg2, ...);

Example:

setTimeout(() => {
  console.log("Hello after 3 seconds");
}, 3000);

BREAK IT DOWN

3000 = 3 seconds (1000 milliseconds = 1 second)

Value Meaning
1000 1 second
2000 2 seconds

🎮 Real Life Example

console.log("Loading...");

setTimeout(() => {
  console.log("Data Loaded");
}, 2000);


// Output:
// Loading...
// (wait 2 seconds)
// Data Loaded

🔁 4. setInterval() (REPEAT FOREVER)

setInterval schedules a function to run repeatedly at specified intervals in milliseconds until it's stopped.

Basic Syntax:

setInterval(function, delay, arg1, arg2, ...);

Example:

setInterval(() => {
  console.log("Tick...");
}, 1000);


// Output:
// Tick...
// Tick...
// Tick...
// (every 1 second)

MENTAL MODEL

setTimeout = "Do later once"

setInterval = "Do later again and again"

5. STOPPING TIMERS (VERY IMPORTANT)

clearTimeout()

const timer = setTimeout(() => {
  console.log("Hello");
}, 3000);

clearTimeout(timer);

Timer canceled❌

clearInterval()

const interval = setInterval(() => {
  console.log("Running...");
}, 1000);

clearInterval(interval);

Repeating stops

6. REAL COUNTDOWN EXAMPLE

let count = 5;

const timer = setInterval(() => {
  console.log(count);

  count--;

  if (count < 0) {
    clearInterval(timer);
    console.log("Done!");
  }
}, 1000);

// Output: 
// 5
// 4
// 3
// 2
// 1
// 0
// Done!

IMPORTANT CONCEPT → CALLBACK FUNCTION

() => {}

This function runs LATER

7. JAVASCRIPT IS NON-BLOCKING

This is VERY important.

Example:

console.log("Start");

setTimeout(() => {
  console.log("Timer");
}, 2000);

console.log("End");

// Output:
// Start
// End
// (wait 2 seconds)
// Timer

WHY?
Because JS says: “I’ll continue my work… timer can wait”

ENGINEERING MINDSET

Timers are NOT magical pauses.
JavaScript schedules them in browser APIs/event loop and continues running other code.

9. COMMON MISTAKES

// Mistake 1:
setTimeout(myFunction(), 1000);
// This runs myFunction immediately instead of after 1 second

// Correct way:
setTimeout(myFunction, 1000);

// Mistake 2:
// Forgetting to clear intervals:
// Causes memory leaks, unwanted behavior and unncecessary resource usage
const interval = setInterval(() => {
  console.log("Running...");
}, 1000);
// If we never call clearInterval(interval), it will keep running forever

// Mistake 3:
// Creating intervals inside intervals
setInterval(() => {
  setInterval(() => {
    console.log("Nested interval");
  }, 1000);
}, 2000);
// This creates a new interval every 2 seconds, leading to exponential growth of intervals and crashing the browser

Auto Counter

const counter = document.querySelector("#counter");

let num = 0;

const interval = setInterval(() => {
  num++;

  counter.textContent = num;

  if (num === 10) {
    clearInterval(interval);
  }
}, 1000);
